Color your terminal without typing whatever the heck `\x1b[31m` is. // Farben 是一个使用类似标记语法的颜色库。为你的终端着色，而无需输入 `\x1b[31m` 这类乱七八糟的东西。 /// Using default features ///: (: use farben::prelude::*) (: style!(\"error\", \"[bold underline red]\") ;: cprintln!(\"[error]error: [/]Something bad happened.\") ;: ...) (: style!(\"warn\",  \"[bold yellow]\") ;: cprintln!(\"[warn]warn: [/]This looks suspicious.\") ;: ...) (: cprintln!(\"[bg:blue fg:white]White on blue!\") ;: let name = \"Razkar\" ;: cprintln!(\"[bold green]Hello, {}!\", name) ;: ...) (: cprintb!(\"[red]This bleeds \") ;: cprintln!(\"into this.\") ;: ...) /// Syntax // 语法 ///: Tags are written as [tag] and apply from that point forward. Multiple tags can be combined in a single bracket: [bold red]. // 标签以 [tag] 的形式编写，并从此处开始应用。多个标签可以组合在单个括号中： [bold red] 。 /// Warning // 警告 ///: Spaces inside ansi() and rgb() are not supported at the moment. // 当前不支持在 ansi() 和 rgb() 内部使用空格。 /// Features // 功能 /// - Markup-like Syntax: Easy to read, write, and powerful when used. // 类似标记的语法：易于阅读、编写，且在使用时功能强大。 /// - Zero required runtime dependencies: Only farben-core as a path dependency, Farben introduces no complicated dependency tree. // 零个运行时依赖项：仅作为路径依赖项 farben-core ，Farben 不会引入复杂的依赖树。 /// - Opt-in Compile-time Processing: Validate and process markup at compile time with no runtime overhead, via the compile feature flag. // 可选编译时处理：通过 compile 功能标志，在编译时验证和处理标记，无需运行时开销。 /// - Complete Toolkit: Named colors, ANSI256, RGB, emphasis styles, style chaining, foreground and background support. // 完整工具集：命名颜色、ANSI256、RGB、强调样式、样式链、前景和背景支持。 /// - Drop-in Print Macros: cprint!, cprintln!, cprintb!, cprintbln! work just like print! and println! but with markup support. Writer variants cwrite!, cwriteln!, cwriteb!, cwritebln! work with any Write implementor. // 即插即用打印宏： cprint! 、 cprintln! 、 cprintb! 、 cprintbln! 就像 print! 和 println! 一样工作，但支持标记。写入器变体 cwrite! 、 cwriteln! 、 cwriteb! 、 cwritebln! 与任何 Write 实现者兼容。 /// - Stderr Variants: All print macros have e variants (ceprint!, ceprintln!, etc.) that target stderr. // 标准错误变体：所有打印宏都有 e 变体（ ceprint! 、 ceprintln! 等），它们指向标准错误。 /// - Bleed Variants: cprintb!, cprintbln!, colorb(), and colorb!() skip the trailing reset, letting styles carry forward across multiple calls. // 溢出变体： cprintb! 、 cprintbln! 、 colorb() 和 colorb!() 跳过尾部重置，使样式在多次调用间得以延续。 /// - User-defined styles: Define your own tags with style!() that expand to any combination of supported tags. // 用户自定义样式：使用 style!() 定义自己的标签，它们会展开为任何支持的标签组合。 /// - Custom style files: Drop in a name.frb.toml file, write 1 line on build.rs and 2 on main.rs, and all styles from that file is set. // 自定义样式文件：放入一个 name.frb.toml 文件，在 build.rs 上写一行，在 main.rs 上写两行，该文件中的所有样式都将被设置。 /// - anstyle Interop: Unlock to the anstyle universe, with the sugar of markup. More on the docs. // anstyle Interop：解锁 anstyle 宇宙，享受标记语的甜头。更多内容请查看文档。"
