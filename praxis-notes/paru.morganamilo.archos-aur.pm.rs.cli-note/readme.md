[src/gh]: https://github.com/morganamilo/paru.git "(GPL-3.0) (Languages: Rust 88.2%, Shell 11.5%, Dockerfile 0.3%) Feature packed AUR helper // 功能丰富的 AUR 助手 /// Paru is your standard pacman wrapping AUR helper with lots of features and minimal interaction. // Paru 是一个标准的 pacman 封装 AUR 助手，具有许多功能且交互最小化。 /// General Tips // 通用技巧 /// - Man pages: For documentation on paru's options and config file see paru(8) and paru.conf(5) respectively. // 手册页：有关 paru 的选项和配置文件的信息，请分别参见 paru(8) 和 paru.conf(5) 。 /// - Color: Paru only enables color if color is enabled in pacman. Enable color in your pacman.conf. // 颜色：Paru 仅在 pacman 启用颜色时才启用颜色。在您的 pacman.conf 中启用 color 。 /// - File based review: To get a more advanced review process enable FileManager with your file manager of choice in paru.conf. // 基于文件的审查：要获得更高级的审查过程，请在 paru.conf 中使用您的首选文件管理器启用 FileManager 。 /// - Flip search order: To get search results to start at the bottom and go upwards, enable BottomUp in paru.conf. // 翻转搜索顺序：要使搜索结果从底部开始并向上显示，请在 paru.conf 中启用 BottomUp 。 /// - Editing PKGBUILDs: When editing PKGBUILDs, you can commit your changes to make them permanent. When the package is upgraded, git will try to merge your changes with upstream's. // 编辑 PKGBUILD：当编辑 PKGBUILD 时，您可以提交您的更改使其永久化。当升级软件包时， git 将尝试将您的更改与上游合并。 /// - PKGBUILD syntax highlighting: You can install bat (. gh:sharkdp/bat.git) to enable syntax highlighting during PKGBUILD review. // PKGBUILD 语法高亮：您可以通过安装 bat 来在 PKGBUILD 审查期间启用语法高亮。 /// - Tracking -git packages: Paru tracks -git package by monitoring the upstream repository. Paru can only do this for packages that paru itself installed. paru --gendb will make paru aware of packages it did not install. // 跟踪 -git 软件包：Paru 通过监控上游存储库来跟踪 -git 软件包。Paru 只能对它自己安装的软件包执行此操作。 paru --gendb 将使 paru 意识到它没有安装的软件包。 /// Debugging // 调试 /// Paru is not an official tool. If paru can't build a package, you should first check if makepkg can successfully build the package. If it can't, then you should report the issue to the maintainer. Otherwise, it is likely an issue with paru and should be reported here. // Paru 不是一个官方工具。如果 paru 无法构建一个软件包，你应该首先检查 makepkg 是否能成功构建该软件包。如果不能，你应该向维护者报告问题。否则，很可能是 paru 的问题，应该在这里报告。"
