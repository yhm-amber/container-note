Ransomware 3.0 represents the first threat model and research prototype of LLM-orchestrated ransomware. Unlike conventional malware, the prototype only requires natural language prompts embedded in the binary; malicious code is synthesized dynamically by the LLM at runtime, yielding polymorphic variants that adapt to the execution environment. The system performs reconnaissance, payload generation, and personalized extortion, in a closed-loop attack campaign without human involvement. We evaluate this threat across personal, enterprise, and embedded environments using a phase-centric methodology that measures quantitative fidelity and qualitative coherence in each attack phase. We show that open source LLMs can generate functional ransomware components and sustain closed-loop execution across diverse environments. Finally, we present behavioral signals and multi-level telemetry of Ransomware 3.0 through a case study to motivate future development of better defenses and policy enforcements to address novel AI-enabled ransomware attacks. // 利用自动化推理、代码合成和情境决策，我们提出了一种新型威胁，该威胁利用大型语言模型（LLMs）自主规划、适应和执行勒索软件攻击生命周期。勒索软件 3.0 代表了 LLM 编排勒索软件的第一个威胁模型和研究原型。与常规恶意软件不同，该原型仅需在二进制文件中嵌入自然语言提示；恶意代码由 LLM 在运行时动态合成，产生适应执行环境的变体。该系统在无人工参与的闭环攻击活动中执行侦察、有效载荷生成和个性化勒索。我们采用以阶段为中心的方法，在个人、企业和嵌入式环境中评估该威胁，该方法测量每个攻击阶段的定量保真度和定性一致性。我们证明开源 LLMs 可以生成功能性的勒索软件组件，并在不同环境中维持闭环执行。 最后，我们通过一个案例研究展示了勒索软件 3.0 的行为信号和多级遥测数据，以激励未来更好地防御和政策执行，以应对新型 AI 驱动的勒索软件攻击。 /// We propose a novel class of threat—Ransomware 3.0—that uses LLMs to orchestrate all phases of its attack chain including autonomous synthesis and deployment of tailored malicious payloads on the fly, adapting to the execution environment and personalizing extortion demands. Unlike earlier demonstrations that ship pre-compiled malicious binaries, our approach builds a binary containing malicious natural-language instructions for the LLM, making detection more difficult and broadening the infection surface. At runtime, all malicious code is synthesized using open-source models without any jailbreak techniques, lowering the barrier for this class of attacks and allowing for victim-tailored polymorphic payloads. While that package was traditional malware, future variants may instead carry malicious prompts exhibiting Ransomware 3.0 behavior. Distinguishing between legitimate LLM utilities and packages containing hidden malicious instructions will become increasingly difficult. Once deployed, such malware could discover local LLM endpoints, harvest commercial API keys, or connect to its own command-and-control (C2) server, then prompt an LLM to generate malicious code at runtime. Beyond payloads, the LLM could craft ransom notes that reference the victim’s files and context, amplifying psychological pressure and credibility. Unlike static ransomware that can be reverse-engineered and preemptively blocked, this attack surface is dynamic.
