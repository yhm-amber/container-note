ncdns域名解析

ncdns is software for accessing .bit domain names. If you want to access .bit domain names, ncdns is what you want to install. ncdns works with most Internet applications (not just web browsers), and uses caching for extra speed (particularly helpful when used with Electrum-NMC). (Technically speaking, ncdns is a Namecoin to DNS bridge. It allows software that speaks the DNS protocol to use Namecoin.)ncdns是访问.bit域名的软件。如果你想访问.bit域名，ncdns就是你要安装的。 ncdns 适用于大多数 Internet 应用程序（不仅仅是 Web 浏览器），并使用缓存来提高速度（在与 Electrum-NMC 一起使用时特别有用）。 （从技术上讲，ncdns 是 Namecoin 到 DNS 的桥梁。它允许使用 DNS 协议的软件使用 Namecoin。）

Installation安装

You can download ncdns at the Downloads page.您可以在下载页面下载 ncdns。

Windows视窗

An installer wizard is available. The ncdns Windows installer will offer to automatically install a Namecoin name lookup client (Namecoin Core, ConsensusJ-Namecoin, or Electrum-NMC) and Dnssec-Trigger:安装程序向导可用。 ncdns Windows 安装程序将提供自动安装 Namecoin 名称查找客户端（Namecoin Core、ConsensusJ-Namecoin 或 Electrum-NMC）和 Dnssec-Trigger：

If you’re missing a dependency for ConsensusJ-Namecoin, you’ll see a notice like this:如果您缺少 ConsensusJ-Namecoin 的依赖项，您会看到如下通知：

Or like this:或者像这样：

To enable installing ConsensusJ-Namecoin, exit the installer wizard, install the relevant dependency, and re-run the installer wizard.要启用安装 ConsensusJ-Namecoin，请退出安装程序向导，安装相关依赖项，然后重新运行安装程序向导。

The ncdns installer wizard also sets up TLS certificate validation for .bit domain names. This will work for any application that uses the Windows certificate verifier. As examples, Chromium and Edge use the Windows certificate verifier, and will therefore work fine for Namecoin TLS out of the box.ncdns 安装程序向导还为 .bit 域名设置 TLS 证书验证。这适用于任何使用 Windows 证书验证程序的应用程序。例如，Chromium 和 Edge 使用 Windows 证书验证程序，因此可以开箱即用地用于 Namecoin TLS。

The ncdns installer wizard also configures Namecoin resolution in Tor Browser, if Tor Browser is installed.如果安装了 Tor 浏览器，ncdns 安装程序向导还会在 Tor 浏览器中配置 Namecoin 解析。

Other OS’s其他操作系统

ncdns plain binaries (without install scripts) are available for most major operating systems. These require installing a Namecoin name lookup client and a DNS resolver (e.g. Dnssec-Trigger) separately, and manually configuring ncdns to integrate with them (see instructions below). ncdns plain binaries are only recommended for advanced users at this time.ncdns 普通二进制文件（没有安装脚本）可用于大多数主要操作系统。这些需要分别安装 Namecoin 名称查找客户端和 DNS 解析器（例如 Dnssec-Trigger），并手动配置 ncdns 以与它们集成（参见下面的说明）。目前仅向高级用户推荐 ncdns 纯二进制文件。

TLS instructions for ncdns on GNU/Linux are at the TLS Client Compatibility page.GNU/Linux 上 ncdns 的 TLS 说明位于 TLS 客户端兼容性页面。

Supplying your own Namecoin node提供您自己的 Namecoin 节点

The ncdns Windows installer will offer to install Namecoin Core, ConsensusJ-Namecoin, or Electrum-NMC, and configure ncdns to use it. However, there are several reasons why you might want to supply your own Namecoin node:ncdns Windows 安装程序将提供安装 Namecoin Core、ConsensusJ-Namecoin 或 Electrum-NMC，并配置 ncdns 以使用它。但是，您可能想要提供自己的 Namecoin 节点的原因有几个：

  • You’re not using Windows.您没有使用 Windows。
  • You want to use a custom Namecoin node implementation.您想要使用自定义 Namecoin 节点实现。
  • You want to run a Namecoin node on a different machine than ncdns.您想在与 ncdns 不同的机器上运行 Namecoin 节点。
  • You want to handle updating your Namecoin node separately from updating ncdns.您希望与更新 ncdns 分开处理更新您的 Namecoin 节点。

If you want to supply your own Namecoin node, you can follow these steps:如果您想提供自己的 Namecoin 节点，您可以按照以下步骤操作：

  1. Make sure that your Namecoin node has the RPC server enabled.确保您的 Namecoin 节点启用了 RPC 服务器。
  2. Open ncdns.conf. On Windows, it’s in the etc/ subdirectory of where you installed ncdns.打开 ncdns.conf。在 Windows 上，它位于安装 ncdns 的 etc/ 子目录中。
  3. Look for the namecoind access section.查找 namecoind 访问部分。
  4. If your Namecoin node isn’t listening on 127.0.0.1, or if it’s listening on a non-default port, uncomment the namecoinrpcaddress line and fill in the IP and port.如果您的 Namecoin 节点没有在 127.0.0.1 上侦听，或者如果它在非默认端口上侦听，请取消注释 namecoinrpcaddress 行并填写 IP 和端口。
  5. If your Namecoin node uses cookie authentication, fill in the path to the cookie file in the namecoinrpccookiepath line. Make sure that the ncdns user has filesystem permissions to read the cookie file. (Be careful not to grant the ncdns user filesystem permissions that it doesn’t need. For example, don’t grant ncdns the ability to read your wallet.)如果您的 Namecoin 节点使用 cookie 身份验证，请在 namecoinrpccookiepath 行中填写 cookie 文件的路径。确保 ncdns 用户具有读取 cookie 文件的文件系统权限。 （小心不要授予 ncdns 用户不需要的文件系统权限。例如，不要授予 ncdns 读取你的钱包的能力。）
  6. If your Namecoin node doesn’t use cookie authentication, comment out the namecoinrpccookiepath line, uncomment the namecoinrpcusername and namecoinrpcpassword lines, and fill in your Namecoin node’s RPC username and password.如果您的 Namecoin 节点不使用 cookie 身份验证，请注释掉 namecoinrpcccookiepath 行，取消注释 namecoinrpcusername 和 namecoinrpcpassword 行，并填写您的 Namecoin 节点的 RPC 用户名和密码。
  7. Restart ncdns. On Windows, you can do this by going to Control Panel -> Administrative Tools -> Services, right-click ncdns, and click Restart.重新启动 ncdns。在 Windows 上，您可以通过转到控制面板 -> 管理工具 -> 服务，右键单击 ncdns，然后单击重新启动来执行此操作。
