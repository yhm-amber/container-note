[src/gh]: https://github.com/89luca89/podman-launcher.git "(GPL-3.0) (Languages: Go 85.3%, Makefile 14.7%) Simple golang wrapper to bundle a podman installation in a single binary // 一个简单的 Go 语言封装器，用于将 podman 安装打包到一个单独的二进制文件中。 /// Podman Launcher // Podman 发射器 /// This project is a simple golang wrapper that uses embed to ship the latest release of gh: mgoltzsche/podman-static.git // 这个项目是一个简单的 Golang 封装器，它使用 embed 来发布 gh: mgoltzsche/podman-static.git 的最新版本。 /// That repo builds and releases all podman components as statically linked binaries this will let us to easily ship the container manager without needing all the dependency resolution of a package manager. // 该仓库会将所有 podman 组件构建并发布为静态链接的二进制文件，这样我们就可以轻松地发布容器管理器，而无需包管理器的所有依赖项解析。 /// This project will take care of shipping the release (together with crun) and setting it up properly in order to work completely from $HOME, and without overlapping with a native podman installation. // 该项目将负责发布版本（以及 crun ），并进行正确配置，使其完全在 $HOME 目录下运行，且不会与其他系统重叠。 使用原生 podman 安装。 /// Rootful podman works (if needed), and will unpack a copy of the binaries in /root for it to work. // Rootful podman 可以工作（如果需要），它会在 /root 目录下解压二进制文件的副本才能正常工作。 /// ## Installation // 安装 /// Download the binary, make it executable and put it in your $PATH // 下载该二进制文件，使其可执行，并将其添加到你的 $PATH 环境变量中。 /// Optionally, you can name it podman in order to make it easier to type/use // 您也可以将其命名为 podman ，以便更方便输入/使用。 /// ## Usage // 用法 /// This launcher is transparent, so you will use it with all podman's flags and so on // 这个启动器是透明的，所以你可以把它和 podman 的所有选项等一起使用。 /// ## Use as lib // 作为库使用 /// You can use the podman-launcher as a library in your project, if you depend on podman and want to embed it as a dependency. // 如果你依赖 podman 并想将其作为依赖项嵌入到你的项目中，你可以将 podman-launcher 作为库使用。 /// You'll need to embed the assets.tar.gz (that you'll find in the release page) in your application, and pass it to the launcher.Config struct for it to work // 您需要将 assets.tar.gz 文件（可在发布页面找到）嵌入到您的应用程序中，并将其传递给 launcher.Config 结构体才能使其正常工作。 /// Example code: // 示例代码： ///: (: package main; import ( _ \"embed\"; \"github.com/89luca89/podman-launcher/pkg/launcher\" ); var assets []byte; func main() { conf := launcher.NewLauncher(\"/home/luca-linux/.podman-launcher\", \"/var/tmp\", assets); command := []string{ \"podman\", \"run\", \"--rm\", \"-ti\", \"alpine:latest\", \"/bin/sh\" }; err := conf.Run(command); if err != nil { if exiterr, ok := err.(*exec.ExitError); ok { os.Exit(exiterr.ExitCode()) } } }) /// ## Upgrade // 升级 /// To update, download the new release, and with the new binary run podman-launcher upgrade to upgrade the embedded podman package. // 要进行更新，请下载新版本，然后使用新二进制文件运行 podman-launcher upgrade 命令。 升级嵌入式 podman 软件包。 /// ## Dependencies // 依赖关系 /// On the system, the only dependencies needed are the one that podman needs. Specifically iptables and ip6tables for the bridge to work (not needed if using host's network namespace). // 该系统仅需 podman 所需的依赖项。具体来说，需要 iptables 和 ip6tables 才能使网桥正常工作（如果使用主机的网络命名空间则不需要）。 /// For rootless setup to work you need newuidmap and newgidmap binaries (usually part of the shadow package) and correctly set the /etc/subuid and /etc/subgid // 要使无根配置正常工作，您需要 newuidmap 和 newgidmap 二进制文件（通常是 shadow 软件包的一部分），并正确设置 /etc/subuid 和 /etc/subgid 文件。 /// ## Use Cases // 用例 /// It's a nice-to-have for systems like the Steamdeck or where you're not allowed to modify the system in any way. // 对于像 Steamdeck 这样的系统，或者不允许以任何方式修改系统的系统来说，这是一个锦上添花的功能。 /// Thought to be a nice fallback container engine option for Distrobox (. gh: 89luca89/distrobox.git) // 被认为是 Distrobox (. gh: 89luca89/distrobox.git) 的一个不错的备用容器引擎选项。"
