[src/gh]: https://github.com/eersnington/jj-navi.git "(MIT) (Languages: Rust 99.0%, JavaScript 1.0%) jj-navi is workspace orchestrator for Jujutsu (jj), built for parallel human and AI agentic workflows // jj-navi 是 Jujutsu (jj) 的工作区协调器，专为人类和 AI 代理并行工作流程而设计。 /// Workspace management for Jujutsu, built for parallel human and AI agent workflows. // 为 Jujutsu 构建的工作区管理，支持人类和 AI 代理并行工作流程。 /// ## The problem // 现存问题 ///: jj workspaces are great for parallel work, but the workflow around it is quite cumbersome: // jj 工作区非常适合并行工作，但围绕它的工作流程却相当繁琐： /// - Paths are unmanaged. jj workspace add ../name works, but paths are arbitrary and easy to forget. // 路径未受管理。虽然 jj workspace add ../name 命令可以正常工作，但路径是任意的，很容易忘记。 /// - Cross-workspace visibility is stale. jj snapshots the current workspace when you run a command, but not the others. So jj log from one workspace can show outdated commits for the rest — files on disk exist, but jj hasn't recorded them yet. // 跨工作区可见性存在问题。jj 会在你运行命令时对当前工作区进行快照，但不会对其他工作区进行快照。因此，从一个工作区 jj log 可能会显示其他工作区的过时提交记录——磁盘上的文件确实存在，但 jj 尚未记录它们。 /// - Cleanup is awkward. Forgetting a workspace does not delete its directory, and deleting a directory does not forget the workspace. There is also no guard against removing the one you are currently in. // 清理工作区很麻烦。 忘记一个工作区并不会删除它的目录，而删除一个目录也不会忘记对应的工作区。而且，也没有任何机制可以防止误删当前所在的工作区。 /// - Switching doesn't switch your shell. jj workspace changes the working copy, not your terminal's current directory. // 切换不会切换你的 shell。jj jj workspace 更改的是工作副本，而不是终端的当前目录。 /// ## What jj-navi does // jj-navi 的功能 ///: jj-navi manages workspace lifecycle: naming, paths, switching, visibility, and cleanup. // jj-navi 管理工作区生命周期：命名、路径、切换、可见性和清理。 /// - switch --create — go to a workspace, creating it at a deterministic path if it doesn't exist // switch --create — 进入一个工作区，如果该工作区不存在，则在指定路径创建它。 /// - list — snapshot each workspace and show path health, diff stats, commit info, and age // list — 为每个工作区生成快照，并显示路径健康状况、差异统计信息、提交信息和提交时间 /// - merge — merge work from another workspace into the current or named workspace // merge — 将其他工作区中的工作合并到当前或指定的工作区中。 /// - remove — forget a workspace and delete its local directory; refuses current workspace // remove — 忘记工作区并删除其本地目录；拒绝当前工作区 /// ## Before and after // 前后对比 /// - Without jj-navi // 无 jj-navi 时 ///: (: jj workspace add ../repo.feature-auth ;: cd ../repo.feature-auth # switch to workspace) (: # ... do your work ...) (: cd ../repo # return to your repo from workspace) (: jj log # stale view of other workspaces ;: jj workspace list # just can see names only) (: jj workspace forget feature-auth # leave workspace ;: rm -rf ../repo.feature-auth # directory left behind) /// - With jj-navi // 有 jj-navi 时 ///: (: navi switch --create feature-auth # create and switch (-c) # one step with no cd need here) (: # ... do your work ...) (: navi switch - # switch to previous workspace ;: navi switch @ # switch to current workspace explicitly) (: navi list # snapshotted, with diff stats and age) (: navi remove feature-auth # leave & remove workspace # asks before deleting the workspace directory) /// ## Notes // 注意 /// - switch can recover from missing jj workspace-path records when it can validate a fallback path // 当 switch 能够验证备用路径时，它可以从丢失的 jj 工作区路径记录中恢复。 /// - switch warns when it falls back to template-based path resolution // 当切换回基于模板的路径解析时， switch 发出警告。 /// - list snapshots healthy workspaces before rendering so parallel changes are visible // 在渲染之前 list 健康工作区的快照，以便并行更改可见。 /// - list reports missing, stale, or not-current workspaces instead of hiding them // list 缺失、过期或非最新工作区的报告，而不是隐藏它们。 /// - list --json exposes structured freshness, diff, and age fields // list --json 会公开结构化的 freshness 、 diff 和 age 字段。 /// - remove forgets a workspace and deletes its directory after confirmation; --yes skips the prompt // remove 会忽略某个工作区，并在确认后删除其目录； --yes 会跳过提示。 /// - Supported shells: bash, zsh // 支持的 shell： bash 、 zsh /// ## Commands // 命令  /// - ws-switch ///: (: navi switch <workspace> # switch to a workspace) (: navi cd <workspace> # alias for switch) (: navi switch - # switch to previous workspace) (: navi switch @ # switch to current workspace explicitly) (: navi switch --create <workspace> // alias: navi switch -c <workspace> # create and switch) (: navi switch --create <workspace> --revision <revset> // alias: navi switch -c <workspace> -r <revset> # create from a revision) /// - ws-list ///: (: navi list // alias: navi ls # human-readable workspace inventory) (: navi list --json // alias: navi list -j) (: navi list --json --compact // alias: navi list -j -c) /// - state-see ///: (: navi doctor [--json] [--compact] // alias: navi doctor [-j] [-c] # diagnose repo, workspace, and shell state) /// - ws-merge ///: (: navi merge --from <workspace> // alias: navi merge -f <workspace> # merge a workspace into the current workspace) (: navi merge --from <workspace> --into <workspace> // alias: navi merge -f <workspace> -i <workspace>) /// - ws-remove ///: (: navi remove <workspace> // alias: navi rm <workspace> # forget a workspace and delete its directory) (: navi remove <workspace> --yes // alias: navi remove <workspace> -y # skip destructive confirmation) /// - sh-init ///: (: navi config shell init <bash|zsh>) (: navi config shell install [--shell <bash|zsh>] // alias: navi config shell install [-s <bash|zsh>])"
