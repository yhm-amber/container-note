Hacking anything with GNU Guix
用 GNU Guix 解决任何事情

by Marius Bakke — Thu 15 September 2022
作者：Marius Bakke — 2022 年 9 月 15 日，星期四

Perhaps my favourite feature of Guix is guix shell. It is one of those tools that I don't know how to do without. Even if you are not ready to use Guix as a package manager (or distro), guix shell alone might be a reason to have Guix installed.
也许我最喜欢的 Guix 特性是 guix shell 。它是我不知道如何没有的那些工具之一。即使您还没有准备好将 Guix 用作包管理器（或发行版）， guix shell 本身也可能是安装 Guix 的原因。

Why?
为什么？

If you are a software developer, system administrator, or anything in between, you have probably experienced a situation where you want to patch some piece of software that you did not write. Either to fix a bug, try an idea you had in the shower, or just have fun.
如果您是软件开发人员、系统管理员或介于两者之间的任何人，您可能遇到过这样一种情况，即您想要修补一些不是您编写的软件。要么修复一个错误，尝试你在淋浴时的想法，要么只是玩得开心。

Then you discover that it needs a mountain of dependencies to build, and that the versions provided by your operating system are too old, or only available on a mixture of PyPI, CPAN, and random repositories. Even if your preferred package manager has all dependencies available, you may not want to install all that just to scratch that itch.
然后你发现它需要大量的依赖项来构建，而且你的操作系统提供的版本太旧，或者只能在 PyPI、CPAN 和随机存储库的混合体上使用。即使您首选的包管理器具有所有可用的依赖项，您可能也不想安装所有这些只是为了解决这个问题。

Enter guix shell. If you are lucky, that project you want to hack on is one of the 21000+ packages available in Guix. Then you can simply clone the repository, navigate to the project in a terminal, and run:
输入 guix shell 。如果幸运的话，您想要破解的项目是 Guix 中可用的 21000 多个包之一。然后你可以简单地克隆存储库，在终端中导航到项目，然后运行：

guix shell --development the-package

That's it. Guix will download everything required to build that package from source, run tests, etc; and make them available for that shell session only. Then you can make, pytest, or whatever you need to try that patch. PYTHONPATH, CPATH, etc are all set up and ready to go.
就是这样。 Guix 将从源代码、运行测试等下载构建该包所需的一切；并使它们仅可用于该 shell 会话。然后您可以 make 、 pytest 或您需要的任何内容来尝试该补丁。 PYTHONPATH 、 CPATH 等都已准备就绪。

Once you exit the shell, the environment is gone, like it never existed. If you want to hack on that project later, simply run the same command again. Now all the dependencies are cached on your system and so creating the environment takes milliseconds.
一旦退出 shell，环境就消失了，就像它从未存在过一样。如果您以后想破解该项目，只需再次运行相同的命令即可。现在所有依赖项都缓存在您的系统上，因此创建环境只需几毫秒。

What if the package is not in Guix, but its dependencies are? You can create arbitrary environments with:
如果包不在 Guix 中，但它的依赖项在 Guix 中怎么办？您可以创建任意环境：

guix shell package1 package2 packageN

Or mix and match:
或者混合搭配：

guix shell -D the-package package1 package2 -D package3

(-D is shorthand for --development)
（ -D 是 --development 的简写）

But what if you don't trust the packages or build systems to not litter your $HOME, steal your passwords, or wipe your data? guix shell has you covered:
但是，如果您不相信这些软件包或构建系统不会乱扔您的 $HOME 、窃取您的密码或擦除您的数据怎么办？ guix shell 你覆盖了吗：

guix shell --container [...]

Now you are in an isolated environment that can only access the current working directory, and with no network access (unless you pass --network). You can make select parts of your file system visible with --expose (read-only) and --share (read/write). This is useful if you just want to try some program, or script, without needing to trust it completely.
现在你处于一个只能访问当前工作目录的隔离环境中，并且没有网络访问权限（除非你通过 --network ）。您可以使用 --expose （只读）和 --share （读/写）使文件系统的选定部分可见。如果您只想尝试某些程序或脚本而不需要完全信任它，这将很有用。

There are more features lurking in guix shell, but these are by far the ones I use the most. I don't even have python available on PATH and just create a shell every time I need it, tailored to the project I want to hack on.
guix shell 中潜伏着更多功能，但这些是迄今为止我使用最多的功能。我什至在 PATH 上都没有可用的 python ，只是在每次需要时创建一个 shell，为我想要破解的项目量身定制。

This blog is proudly powered by a guix shell -m manifest.scm -- haunt build step.
此博客自豪地由 guix shell -m manifest.scm -- haunt build 步骤提供支持。

Happy hacking!
玩得开心！




"Guix drops" part 1: package transformations
“Guix drops”第 1 部分：包转换

by Marius Bakke — Mon 14 November 2022
作者：Marius Bakke——2022 年 11 月 14 日星期一

This is the first part of a series where I'll showcase advanced features of Guix that may not be known to the average user.
这是系列的第一部分，我将展示普通用户可能不知道的 Guix 高级功能。

They are inspired by the "Nix pills" series. I'm calling them "drops" as some pills are hard to swallow, and because I'm not a total fraud. The best medicines are often liquid, too.
他们的灵感来自“Nix pills”系列。我称它们为“滴剂”，因为有些药片很难吞咽，而且我不是彻头彻尾的骗子。最好的药物通常也是液体的。

One of the coolest features of Guix is the ability to transform packages "on the fly", directly from the command line. Software is, as its name implies, supposed to be malleable, unlike hardware and some humans.
Guix 最酷的特性之一是能够直接从命令行“即时”转换包。顾名思义，软件应该是可塑的，不像硬件和某些人类。

Many transformation options are available, I'll demonstrate some of them here.
有许多转换选项可用，我将在这里演示其中的一些。

--with-source is useful when you want to build a package with a different source code than the one in Guix. I previously wrote about how guix shell is great for software development: --with-source makes it easy to also install your patched software, and test other packages against the patched version.
--with-source 当你想构建一个源代码与 Guix 不同的包时很有用。我之前写过 guix shell 对软件开发有何好处： --with-source 还可以轻松安装已打补丁的软件，并针对已打补丁的版本测试其他软件包。

A real world example is trying software that I hack on:
一个真实世界的例子是尝试我破解的软件：

guix shell zabbix-cli \
  --with-source=zabbix-cli=$HOME/src/zabbix-cli \
  -- zabbix-cli

This will build zabbix-cli with my local source checkout and enter a new shell environment with the patched zabbix-cli available. The -- zabbix-cli at the end instructs guix shell to also run the patched zabbix-cli executable directly instead of just giving that boring Bash prompt waiting for my input.
这将使用我的本地源签出构建 zabbix-cli 并进入一个新的 shell 环境，其中包含可用的补丁 zabbix-cli 。最后的 -- zabbix-cli 指示 guix shell 也直接运行修补后的 zabbix-cli 可执行文件，而不是只给出无聊的 Bash 提示符等待我的输入。

Another cool transformation option is --with-latest. Let's say you are a user of zabbix-cli and the package in Guix is older than the latest release (shame on me for that, as a contributor to both projects!). You can simply instruct guix shell (or guix install, guix package, etc) to use the latest version:
另一个很酷的转换选项是 --with-latest 。假设您是 zabbix-cli 的用户，并且 Guix 中的包比最新版本更旧（我为此感到羞耻，作为这两个项目的贡献者！）。您可以简单地指示 guix shell （或 guix install 、 guix package 等）使用最新版本：

guix shell zabbix-cli --with-latest=zabbix-cli

Guix will download the latest tagged release and build it on the fly. All transformations are cached, so if you run the same command again it does not have to be built anew (unless an even newer version is released, of course, at which point Guix will hopefully consider it).
Guix 将下载最新的标记版本并即时构建它。所有转换都被缓存，因此如果您再次运行相同的命令，则不必重新构建它（除非发布了更新的版本，当然，届时 Guix 希望会考虑它）。

This can also be used to try newer versions of dependencies. For example, if I want to check whether zabbix-cli works with the latest requests package:
这也可用于尝试更新版本的依赖项。例如，如果我想检查 zabbix-cli 是否适用于最新的 requests 包：

guix shell zabbix-cli --with-latest=python-requests

Now Guix will fetch the latest version of python-requests and build zabbix-cli against it (and run tests, etc). Neat!
现在 Guix 将获取最新版本的 python-requests 并针对它构建 zabbix-cli （并运行测试等）。整洁的！

For packages that are built from a Git checkout, you can also use specific commits or branches:
对于从 Git 签出构建的包，您还可以使用特定的提交或分支：

guix shell zabbix-cli --with-commit=zabbix-cli=abc123 \
  --with-branch=python-requests=late-bugfixes-2022 \
  --with-source=python=within

Here we also build the Python interpreter from a hypothetical within directory which presumably contains some long overdue performance improvements.
在这里，我们还从假设的 within 目录构建 Python 解释器，该目录可能包含一些早就应该进行的性能改进。

Creativity is the limit for these transformations; I often use them along with git bisect, for example to track down which change in SQLite lead to a test failure in NSS.
创造力是这些转变的极限；我经常将它们与 git bisect 一起使用，例如跟踪 SQLite 中的哪些更改导致 NSS 中的测试失败。

The amount of effort that has been put into making Guix such a developer-friendly tool is humbling! I wish I could take credit for some of it, but that honor is not mine to bear. I'm just a happy user of these parts of the Guix codebase.
为使 Guix 成为这样一个对开发人员友好的工具所付出的努力令人羞愧！我希望我能把其中的一些归功于自己，但这种荣誉不是我能承受的。我只是 Guix 代码库这些部分的快乐用户。




"Guix drops" part 2: garbage collection
“Guix drops”第 2 部分：垃圾回收

by Marius Bakke — Mon 28 November 2022
作者：Marius Bakke——2022 年 11 月 28 日星期一

This is the second post in a series showcasing advanced features of Guix. You can read the first part here.
这是展示 Guix 高级功能的系列文章中的第二篇。你可以在这里阅读第一部分。

As you may know, Guix upgrades are transactional: the new state is computed up-front and "switched to" in one atomic operation. Unlike some other package managers (and real life for that matter!), it is not possible to get into a partially completed upgrade situation.
如您所知，Guix 升级是事务性的：新状态是预先计算的，并在一个原子操作中“切换到”。与其他一些包管理器（以及与此相关的现实生活！）不同，不可能进入部分完成的升级情况。

Also unlike real life, a bad transaction can always be rolled back. If you messed up your system somehow, the resolution is a simple guix system roll-back away. No need to say "I'm sorry, we need to start over" to your mom who just wanted to install that totally legitimate looking casino game.
同样与现实生活不同的是，错误的交易总是可以回滚。如果你以某种方式搞砸了你的系统，解决办法很简单 guix system roll-back 。无需对只想安装看起来完全合法的赌场游戏的妈妈说“对不起，我们需要重新开始”。

Naturally this luxury uses more disk space than a traditional happy-go-lucky operating system. That's where guix gc comes in. When called without any arguments, guix gc will delete all store items (i.e. packages) that are not in use by active system generations (or user profiles).
自然地，这种奢侈使用的磁盘空间比传统的随遇而安的操作系统要多。这就是 guix gc 的用武之地。当不带任何参数调用时， guix gc 将删除所有未被活动系统生成（或用户配置文件）使用的存储项（即包）。

However that alone won't free up disk space from old generations that are no longer useful. To do that you first need to run guix system delete-generations to make those eligible for garbage collection:
然而，仅此一项并不能释放不再有用的老年代的磁盘空间。为此，您首先需要运行 guix system delete-generations 以使那些符合垃圾收集条件：

$ sudo guix system delete-generations 3m
deleting /var/guix/profiles/system-122-link
deleting /var/guix/profiles/system-123-link
deleting /var/guix/profiles/system-124-link
building /gnu/store/r8g38hm98wcz5vn46ifv1ylrs525rvqp-grub-locales.drv...
building /gnu/store/0764m04djxrn0b85nzdmp8chfrci0jdj-grub.cfg.drv...
building /gnu/store/pgrr3q31z4wq3bwrc715aaynglrq7zmh-install-bootloader.scm.drv...

This deleted all generations older than three months and also removed the entries from the bootloader menu. All gone, just like that! If you don't pass a date argument, everything except the current generation will be deleted. Better make sure you won't regret that later (like I have!).
这删除了所有超过三个月的世代，并从引导加载程序菜单中删除了条目。都过去了，就这样！如果您不传递日期参数，则除当前代之外的所有内容都将被删除。最好确保您以后不会后悔（就像我一样！）。

To clear old generations from your user profile, use guix package --delete-generations:
要从您的用户配置文件中清除老年代，请使用 guix package --delete-generations ：

$ guix package --delete-generations=3m
deleting /var/guix/profiles/per-user/marius/guix-profile-35-link
deleting /var/guix/profiles/per-user/marius/guix-profile-36-link
deleting /var/guix/profiles/per-user/marius/guix-profile-37-link
deleting /var/guix/profiles/per-user/marius/guix-profile-38-link

Notice that it only deletes links, not actual packages. If we now run guix gc, the packages inside those profile generations will be deleted (as long as they are not referenced by other generations).
请注意，它只删除 links ，而不是实际的包。如果我们现在运行 guix gc ，这些配置文件生成中的包将被删除（只要它们不被其他生成引用）。

Conveniently guix gc can delete old generations and clear up disk space in one go. This also includes generations from guix pull and guix home:
方便 guix gc 可以一次性删除老年代，清理磁盘空间。这还包括来自 guix pull 和 guix home 的生成：

$ guix gc --delete-generations=3m
deleting /var/guix/profiles/per-user/marius/current-guix-14-link
deleting /var/guix/profiles/per-user/marius/current-guix-15-link
deleting /var/guix/profiles/per-user/marius/current-guix-16-link
deleting /var/guix/profiles/per-user/marius/guix-home-52-link
deleting /var/guix/profiles/per-user/marius/guix-home-53-link
deleting /var/guix/profiles/per-user/marius/guix-home-54-link
finding garbage collector roots...
[... lots of output about deleted packages ...]
deleting `/gnu/store/trash'
deleting unused links...
note: currently hard linking saves 627740.99 MiB
guix gc: freed 80,173.42858 MiBs

If you run this command as root or with sudo, it will clear old generations from all users which is useful on a multi-user system.
如果您以 root 身份或使用 sudo 运行此命令，它将清除所有用户的老年代，这在多用户系统上很有用。

For developers who just want to free up some space, but not necessarily delete everything (because they will inevitably need some of the unreferenced packages later), there are handy flags to only control how much to delete.
对于只想释放一些空间但不一定要删除所有内容（因为他们以后不可避免地需要一些未引用的包）的开发人员，可以使用方便的标志来控制删除多少。

For example, to clear up 40GiB for that other operating system you want to try in a VM:
例如，要为您要在 VM 中尝试的其他操作系统清理 40GiB：

guix gc -C 40G

Or if you want to make 100GiB available on the system:
或者，如果您想在系统上提供 100GiB：

guix gc -F 100G

This differs from -C in that it will stop once the free disk space reaches 100GiB. If you already have 100GiB available space, the command does nothing.
这与 -C 的不同之处在于它会在可用磁盘空间达到 100GiB 时停止。如果您已经有 100GiB 可用空间，则该命令不执行任何操作。

The latter is useful as a cron job to make sure the store does not take up the whole disk, while not deleting all those builds you worked so hard for (but did not install to a profile).
后者作为 cron 作业很有用，可确保存储不会占用整个磁盘，同时不会删除您辛勤工作的所有构建（但未安装到配置文件）。




"Guix drops" part 3: G-expressions
“粉笔滴”第 3 部分：G 表达式

by Marius Bakke — Tue 29 November 2022
作者：Marius Bakke — 2022 年 11 月 29 日，星期二

This is the third post in a series showcasing advanced features of Guix. Previous posts can be found below:
这是展示 Guix 高级功能的系列文章的第三篇。以前的帖子可以在下面找到：

  • package transformations.
    包转换。
  • garbage collection.
    垃圾收集。

If you've used Guix a while you may have heard of G-expressions. You may have even used them. But what are they? The inetd system test says it best:
如果您使用过 Guix 一段时间，您可能听说过 G 表达式。你甚至可能用过它们。但它们是什么？ inetd 系统测试说得最好：

Like an S-expression but with a G.
类似于 S 表达式，但带有 G。

Even as a seasoned Guix developer, when G-expressions were introduced it took me a long time before having that "gee, I'm starting to understand how this all works" moment. Becoming proficient at it took even longer.
即使作为一名经验丰富的 Guix 开发人员，当 G 表达式被引入时，我也花了很长时间才拥有“哎呀，我开始理解这一切是如何工作的”时刻。精通它需要更长的时间。

"Give me a break, I've only been working here five years!"
“放过我吧，我在这里工作才五年！”

The academically inclined may want to check out this paper for an in-depth introduction. I'll try to break the concepts down a little for laypeople or learning challenged (like myself).
有学术倾向的人可能想查看这篇论文以获得更深入的介绍。我将尝试为外行或学习受到挑战的人（比如我自己）分解一些概念。

(Be sure to also check out the Guix manual)
（一定要查看 Guix 手册）

I think of G-expressions as a scripting language with the usual niceties of Guix: reproducible and cached computations. It's an extension of the Scheme language that brings the power of all 20k+ packages in Guix with it.
我认为 G 表达式是一种脚本语言，具有 Guix 的常见优点：可重现和缓存计算。它是 Scheme 语言的扩展，它带来了 Guix 中所有 20k+ 包的强大功能。

Here is an example from my system configuration:
这是我的系统配置的示例：

(use-modules (guix gexp)
             (gnu artwork)
             (gnu packages inkscape))

(define background
  (computed-file
   "guix-background.png"
   #~(begin
       (let ((inkscape #+(file-append inkscape "/bin/inkscape"))
             (backgrounds #$(file-append %artwork-repository "/backgrounds")))
         (system* inkscape (string-append "--export-filename=" #$output)
                  (string-append backgrounds "/guix-silver-checkered-16-9.svg")
                  "-w" "2560" "-h" "1440")))))

This computes a nice PNG file from the raw SVG logo found in the guix artwork repository.
这从 guix 图稿存储库中找到的原始 SVG 徽标计算出一个漂亮的 PNG 文件。

This computed PNG is then used in my window manager configuration:
然后在我的窗口管理器配置中使用这个计算出的 PNG：

(define %sway-config
  (mixed-text-file
   "sway-config"
[...]
### Output configuration
#
# Wallpaper
output * bg " background " fill

mixed-text-file is an N-argument procedure that creates a text file from the strings and variables passed as arguments. A more gexp-y way to declare the configuration would be to use computed-file like so:
mixed-text-file 是一个 N 参数过程，它根据作为参数传递的字符串和变量创建文本文件。声明配置的更 gexp-y 方式是使用 computed-file ，如下所示：

(define %sway-config
  (computed-file
   "sway-config"
   #~(call-with-output-file #$output
[...]
# Wallpaper
output * bg #$background fill

In both cases the result is the same: the background we computed above is inserted into the configuration file with the full /gnu/store/... file name.
在这两种情况下，结果是相同的：我们上面计算的背景被插入到具有完整 /gnu/store/... 文件名的配置文件中。

If the SVG file (or Inkscape) is updated, the background will be computed anew, fully transparent to the configuration. Otherwise the cached result is used.
如果 SVG 文件（或 Inkscape）更新，背景将重新计算，对配置完全透明。否则使用缓存的结果。

This makes for a very powerful scripting language where all the programs you need are "already installed", and each computation is cached. If any of the inputs (e.g. packages) to the script change, the cache is automatically invalidated.
这形成了一种非常强大的脚本语言，其中您需要的所有程序都“已安装”，并且每个计算都被缓存。如果脚本的任何输入（例如包）发生变化，缓存将自动失效。

The gexp machinery works "out of the box" in the context of guix build, inside manifests, etc. It took me a while to figure out how to access this in general-purpose scripts. Here is a toy example that calculates the factorial of the given argument and saves the result in the store:
gexp 机制在 guix build 的上下文、内部清单等中“开箱即用”。我花了一段时间才弄清楚如何在通用脚本中访问它。这是一个玩具示例，它计算给定参数的阶乘并将结果保存在商店中：

#!/usr/bin/env -S guile --no-auto-compile
;; -*- mode: Scheme;-*-
!#

(use-modules (guix gexp)
             (guix derivations)
             (guix store))

(define (calculate-factorial)
  (let ((argument (cadr (command-line))))
    (gexp->derivation (string-append "factorial-of-" argument)
      #~(begin
          (use-modules (ice-9 format))
          (define (factorial n)
            (if (zero? n)
                1
                (* n (factorial (- n 1)))))
          (let ((result (factorial (string->number #$argument))))
            (call-with-output-file #$output
              (lambda (port)
                (format port "~d~%" result))))))))

(let* ((store (open-connection))
       (drv (run-with-store store (calculate-factorial)))
       (output (derivation->output-path drv)))
  (build-derivations store (list drv))
  (format #t "~a~%" output))

It can be run with guile toy.scm or made executable and run directly:
它可以使用 guile toy.scm 运行，也可以使其可执行并直接运行：

$ ./toy.scm 42
/gnu/store/brmr07dx7mhrm9f0wfic2rvx22dgyrzz-factorial-of-42
$ cat $(!!)
1405006117752879898543142606244511569936384000000000

Calculating the factorial of 25000 took 4 seconds on my machine. Running the same command again hit the cache and took 0.7 seconds.
在我的机器上计算 25000 的阶乘需要 4 秒。再次运行相同的命令会命中缓存并耗时 0.7 秒。

It wasn't for nothing
这不是无缘无故的

by Marius Bakke — Wed 17 August 2022
作者：Marius Bakke — 2022 年 8 月 17 日，星期三

Several friends and family members have asked me why I spend so much of my free time developing Guix. I don't always have a good answer, but after six years I'm starting to get a sense of it.
几个朋友和家人问我为什么要花这么多空闲时间开发 Guix。我并不总是有一个好的答案，但六年后我开始对它有所了解。

First of all, it's fun. It's fun to make my own operating system. I can design it exactly how I want it, from the inner workings to the user interface.
首先，它很有趣。制作我自己的操作系统很有趣。我可以按照自己的意愿设计它，从内部运作到用户界面。

It's also rewarding. While I haven't been as active on the patch tracker as I'd like lately, it gives a sense of awe and accomplishment to aid new contributors from beginners to "professionals". It's also something of a dopamine rush to perform huge migrations like switching to a new C compiler or Python version upgrades (these activities feel more like an addiction than a hobby, but rewarding nonetheless).
这也是有益的。虽然我最近没有像我希望的那样积极参与补丁跟踪器，但它给人一种敬畏和成就感，可以帮助从初学者到“专业人士”的新贡献者。这也是执行大规模迁移的多巴胺冲动，例如切换到新的 C 编译器或 Python 版本升级（这些活动感觉更像是一种瘾而不是一种爱好，但仍然是有益的）。

The work feels important. Guix occupies an underserved niche in the operating system "market". Namely one that offers full reproducibility, transparency, and provenance, of every package, and every operating system generation. It also integrates a neat scripting language where you can string together any package into one deterministic script. Think of it like a shell script where every command is lazily instantiated by the package manager on demand. You don't even have to use Scheme, you can write in your favorite language and run it through guix shell and get the same deterministic guarantees. If you note the Guix revision (as given by guix describe), you can use guix time-machine to reproduce the exact same environment in the future, bit by bit (at least in theory).
工作感觉很重要。 Guix 在操作系统“市场”中占据了一个服务不足的利基市场。即提供每个软件包和每一代操作系统的完全可再现性、透明度和出处。它还集成了一种简洁的脚本语言，您可以在其中将任何包串在一起成为一个确定性脚本。把它想象成一个 shell 脚本，其中每个命令都由包管理器按需延迟实例化。您甚至不必使用 Scheme，您可以用您喜欢的语言编写并通过 guix shell 运行它并获得相同的确定性保证。如果您注意到 Guix 修订版（由 guix describe 给出），您可以在将来使用 guix time-machine 一点一点地（至少在理论上）重现完全相同的环境。

Guix is pure. It's refreshing to know that I can take any package and reuse its source code for my own purposes without having to worry about commercial restrictions or redistribution. It's reassuring to know that all packages are bootstrapped and not just some random binary blobs. Guix is also immensely hackable, it's easy to add extra packages or services without submitting them, either by creating a channel or "inlining" straight in the OS configuration.
Guix 是纯粹的。令人耳目一新的是，我可以使用任何包并为我自己的目的重用其源代码，而不必担心商业限制或重新分发。令人欣慰的是，所有包都是自举的，而不仅仅是一些随机的二进制 blob。 Guix 也非常易于破解，无需提交即可轻松添加额外的包或服务，方法是创建通道或直接在操作系统配置中“内联”。

But most importantly, Guix saved my life. When I first started contributing to Guix I did not have any direction in life, any sense of purpose or belonging. I'd been using GNU/Linux for a long time, been a "power user" of Debian, Gentoo, and NixOS, but did not contribute much to any either because packaging was too much work, or I didn't enjoy the community, or I found the packaging standards "too low". With Guix I finally felt home, it ticked all the boxes, the language is a joy to work with (even though it felt weird in the beginning), and it also desperately needed developers.
但最重要的是，Guix 救了我的命。当我刚开始为 Guix 做贡献时，我的生活没有任何方向，没有任何目的感或归属感。我已经使用 GNU/Linux 很长时间了，是 Debian、Gentoo 和 NixOS 的“高级用户”，但没有对任何一个做出太多贡献，因为打包工作太多，或者我不喜欢社区，或者我发现包装标准“太低”。有了 Guix，我终于有了家的感觉，它满足了所有的要求，使用这种语言是一种乐趣（尽管一开始感觉很奇怪），而且它也迫切需要开发人员。

I know some are disappointed that I have prioritized a hobby over doing "normal" things like chasing an exciting career or strengthening my social network. But without Guix I would still be a confused alcoholic with daddy issues crashing on someones couch. I have now been with Guix for longer than I held any job, and am finally feeling some stability in life; and a desire to explore other creative outlets such as blogging (hi!), cooking, growing sustainable foods; and healthy activities like climbing and skiing.
我知道有些人很失望，因为我将爱好置于追求令人兴奋的职业或加强社交网络等“正常”事情之上。但是如果没有 Guix，我仍然会是一个困惑的酒鬼，因为父亲的问题会在某人的沙发上崩溃。我现在在 Guix 工作的时间比我从事任何工作的时间都长，并且终于感受到了生活的稳定；以及探索其他创意渠道的愿望，例如写博客（嗨！）、烹饪、种植可持续食品；和健康的活动，如登山和滑雪。

None of this would be possible without Guix, and the (occasionally reluctant) support from my awesome friends and family. So thank you. It wasn't for nothing. Rather the opposite.
如果没有 Guix，以及我很棒的朋友和家人的（偶尔不情愿的）支持，这一切都不可能实现。所以谢谢。这不是无缘无故的。恰恰相反。




I'm a 12 year old boy in a 36 year old body
我是36岁身体里的12岁男孩

by Marius Bakke — Fri 11 November 2022
作者：Marius Bakke——2022 年 11 月 11 日，星期五

I previously shared a post detailing why Guix meant so much to me and was surprised and moved by all the feedback.
我之前分享了一篇文章，详细说明了为什么 Guix 对我如此重要，并且对所有反馈感到惊讶和感动。

Since then I went on something of a healing retreat (more like a hermitage): I shut out pretty much everyone in my life and took a deep-dive into topics of healing from complex trauma.
从那以后，我开始了某种疗愈疗养（更像是隐居）：我几乎把生活中的每一个人都拒之门外，并深入探讨了复杂创伤的疗愈话题。

Complex PTSD, also known as childhood PTSD, comes in many flavors. It can be generally described as one or more behaviors adapted as a response to perceived threats in the environment over an extended period of time.
复杂的 PTSD，也称为儿童 PTSD，有多种类型。它通常可以被描述为一种或多种行为，作为对长时间内感知到的环境威胁的响应。

Our brains are beautiful survival machines that will do whatever they can to stay safe and connected. Children growing up in dysfunctional families often pick up behaviors tailored to get their caretakers attention and support. For example, if your parents are neglectful, a strategy could be to make a lot of noise and drama when you need something.
我们的大脑是美丽的生存机器，会尽其所能保持安全和联系。在功能失调的家庭中长大的孩子通常会采取专门为获得照顾者的关注和支持而量身定制的行为。例如，如果你的父母疏忽大意，一个策略可能是在你需要什么的时候大吵大闹。

While such behaviours were necessary for survival as kids, they no longer serve us in the adult world. Instead they become detrimental to building healthy relationships.
虽然这些行为对于孩提时代的生存是必要的，但在成人世界中它们不再为我们服务。相反，它们不利于建立健康的关系。

There was nothing particularly sinister about my childhood, except that my primary (and for a long time only) caretaker was not able to provide adequate care and support, and had serious anger management issues (but was not physically violent).
我的童年并没有什么特别不祥的地方，除了我的主要（而且是很长一段时间内）的看护人无法提供足够的照顾和支持，并且有严重的愤怒管理问题（但没有身体暴力）。

As a result, I learned to sacrifice my own needs for the sake of keeping peace. I could not show emotions because it could trigger a rage fit. The concept of self worth was entirely foreign to me until recently. I had virtually zero boundaries, and never learned to make deep relations; instead I would adapt my personality and cling on to anyone who would let me get close.
结果，我学会了为了保持和平而牺牲自己的需要。我无法表达情绪，因为这可能会引发愤怒。直到最近，自我价值的概念对我来说还是完全陌生的。我几乎没有界限，也从未学会建立深厚的关系；相反，我会调整自己的性格，紧紧抓住任何能让我亲近的人。

Healing from complex trauma is often described as a healing journey. Everyones journey is different, as no two people have the exact same experience. I had to go into solitude because my problems involved breaking down deeply ingrained (social) behavioral patterns. Whenever I went out in public the persona I had adapted to survive would keep showing up and reinforcing itself.
从复杂的创伤中康复通常被描述为一次康复之旅。每个人的旅程都是不同的，因为没有两个人拥有完全相同的经历。我不得不独处，因为我的问题涉及打破根深蒂固的（社会）行为模式。每当我出现在公共场合时，我为生存而适应的角色都会不断出现并自我强化。

One treatment that was effective for me was to visualize "going back in time" to the age where the behavior or trauma I needed to work with started. To connect with that "inner child" through meditation or hypnosis (there are excellent resources on YouTube) and give them the support and affection they needed.
一种对我有效的治疗方法是想象“回到过去”到我需要处理的行为或创伤开始的年龄。通过冥想或催眠（YouTube 上有极好的资源）与“内在小孩”建立联系，并给予他们所需的支持和关爱。

The title of this post alludes to a perceived "emotional age". I have been working largely chronologically through periods of my life, starting at infancy(!). My latest breakthrough was connecting with the careless 8-10 year old in me that learned how to lie and manipulate his way out of just about any situation. That was a difficult one, one I could not "reach" through the usual means of offering a hug, someone to talk to, or play with. What ultimately worked was to put on a cartoon show for kids and imagining I was watching with him, and he eventually "calmed down".
这篇文章的标题暗示了一个被认为是“情绪化的时代”。从婴儿期（！）开始，我一直在按照我生命中各个时期的时间顺序进行工作。我最近的突破是与我内心粗心的 8-10 岁孩子建立联系，他学会了如何撒谎和操纵自己的方式摆脱几乎任何情况。那是一个困难的过程，我无法通过提供拥抱、交谈或玩耍的通常方式来“触及”。最终奏效的是为孩子们放一个卡通节目，想象我和他一起看，他最终“平静下来”。

The last paragraph probably sounds nonsensical if you haven't done inner child work before. It's really about imagining that you are meeting with a younger version of yourself, at a particular age, and reaching out with love and empathy. Ask what they need to feel safe, to feel loved, to feel heard; to comfort them. I usually use a guided meditation to get going and have saved some of my favorites in this playlist.
如果你以前没有做过内在孩子的工作，最后一段可能听起来很荒谬。这真的是想象你正在与一个特定年龄的年轻版本的自己会面，并用爱和同理心伸出援手。询问他们需要什么来感到安全、被爱、被倾听；安慰他们。我通常使用引导式冥想来开始，并在此播放列表中保存了一些我最喜欢的。

Having to put on an actual cartoon and imagining that I was watching with a younger me was next-level. I wonder what teen me will need; maybe playing with cool toys and going to social events? Perhaps I'll write about that some day.
不得不穿上一部真正的卡通片，并想象我正在和一个更年轻的我一起看，这是下一个层次。我想知道我需要什么样的青少年；也许玩很酷的玩具并参加社交活动？也许有一天我会写这个。

While I have deliberately and systematically gone through my life in chronological order, healing from childhood trauma is really a lifelong journey and I'll undoubtedly go "back and forth" as I discover and become aware of more parts of my "shadow", the unconscious parts driving my behavior. It can feel like peeling layers off an onion: once I have access to the feeling that drives a nervous system response, I can reach other feelings that have been masked by that particular behavior.
虽然我有意且系统地按时间顺序度过了我的一生，但从童年创伤中治愈确实是一生的旅程，当我发现并意识到我的“影子”的更多部分时，我无疑会“来回”无意识的部分驱动着我的行为。感觉就像剥洋葱一样：一旦我获得了驱动神经系统反应的感觉，我就可以获得被该特定行为掩盖的其他感觉。

(side note: Troll Hunters is surprisingly good, who knew Guillermo del Toro made children friendly content! I'll probably keep watching with my 10 year old self; maybe he'll even talk to me eventually.)
（旁注：Troll Hunters 出奇的好，谁知道 Guillermo del Toro 制作了儿童友好的内容！我可能会继续和我 10 岁的自己一起看；也许他最终会和我说话。）

Owning my truth
拥有我的真相

by Marius Bakke — Sat 12 November 2022
作者：Marius Bakke — 2022 年 11 月 12 日，星期六

(This is yet another mental health post; if you are here for Guix please stay tuned, there will be technical content again soon.)
（又是一篇心理健康贴，如果你是来Guix的请继续关注，很快又会有技术内容。）

At school we learn about the "fight, flight or freeze" states of the nervous system. Most, if not all, vertebrates exhibit one of these responses when facing a threat.
在学校，我们了解神经系统的“战斗、逃跑或冻结”状态。大多数（如果不是全部）脊椎动物在面临威胁时都会表现出其中一种反应。

Social animals such as humans have another little-known threat response called "fawning". It is a kind of "people pleasing" state: suppressing ones own needs to serve others.
人类等群居动物还有另一种鲜为人知的威胁反应，称为“讨好”。这是一种“讨人喜欢”的状态：压抑自己的需要去服务他人。

Fawning can be commonly found in abuse victims. If you have an abusive partner, it makes sense to tune in to that other persons preferences and act out whatever you think causes the least amount of harm. It is a rather brilliant strategy.
阿谀奉承在虐待受害者身上很常见。如果你有一个虐待伴侣，调整对方的偏好并采取你认为造成最小伤害的任何行为是有意义的。这是一个相当出色的策略。

Yesterday I wrote about calming my deceitful "inner child". Making peace with that part of me has allowed me to uncover some of the underlying fawning dynamics that caused me to start lying and manipulating at a young age.
昨天我写了一篇关于安抚我骗人的“内心小孩”的文章。与我的那部分和平相处让我发现了一些潜在的讨好动力，这些动力导致我在年轻时开始撒谎和操纵。

At that point I was already a world-class people pleaser, but it was not enough to prevent my caretaker from making me feel miserable. In retrospect I realize nothing could ever be good enough: he was just a very angry person. But the innocent young mind kept developing new strategies to try to make him happy.
那时我已经是世界级的讨人喜欢者了，但这还不足以阻止我的看护人让我感到痛苦。回想起来，我意识到没有什么比这更好的了：他只是一个非常生气的人。但是天真幼小的头脑不断地想出新的策略来让他开心。

In late teens I realized manipulating other people was obviously not a good way to behave and managed to largely overcome that habit by actively training myself to say "the truth" no matter what. It took several years: in the beginning it was strange to "observe" myself lying to others about even trivial things. I knew it was wrong yet kept doing it anyway!
在十几岁的时候，我意识到操纵他人显然不是一个好的行为方式，并且通过积极训练自己无论如何都要说“真相”，从而在很大程度上克服了这种习惯。花了几年时间：一开始，“观察”到自己甚至在琐碎的事情上对别人撒谎是很奇怪的。明明知道错了还是坚持！

("the truth" was mainly just stating facts: I did not learn to speak my truth)
（“真相”主要是陈述事实：我没有学会说出我的真相）

I did not recognize my people pleasing behavior until earlier this year. Even after becoming aware I did not know how to stop it. I would again "observe" myself bending over backwards to make people around me "happy", even at great cost to my own needs and integrity.
直到今年早些时候，我才意识到我的员工有取悦他人的行为。即使在意识到之后我也不知道如何阻止它。我会再次“观察到”自己竭尽全力让我周围的人“快乐”，即使我自己的需要和完整性付出了巨大的代价。

Brains are remarkably good at automating tasks. While fawning is a sort of nervous system response to a threatening social situation, it quickly becomes an automatic habit when you live with abusive partners over time. Like most habits, the best way to change it is by replacing the routine with something else.
大脑非常擅长自动执行任务。虽然阿谀奉承是一种神经系统对威胁性社交情况的反应，但随着时间的推移，当你与施虐者一起生活时，它很快就会成为一种自动习惯。像大多数习惯一样，改变它的最好方法是用其他东西代替常规。

With fawning, the "cue" is any situation where you have to make decisions that affect both yourself and others. The "routine" is to consider what is best for your abusive partner -- or for chronic fawners such as myself -- what is best for anyone except myself. The "reward" is not getting abused.
对于奉承，“提示”是指您必须做出影响自己和他人的决定的任何情况。 “常规”是考虑什么对你的虐待伴侣最好——或者对于像我这样的长期奉承者——什么对除了我自己以外的任何人最好。 “奖励”没有被滥用。

The key to overcome fawning then is to recognize own needs and bring them in the cognitive loop. Consider what feels good (or bad) instead of just habitually do or say what you think the other person wants to hear / minimize conflict.
克服阿谀奉承的关键是认清自己的需求，并将其纳入认知循环。考虑什么感觉好（或坏），而不是只是习惯性地做或说你认为对方想听的话/尽量减少冲突。

This is a completely new territory for me and feels both scary and exciting. For the first time in over 30 years I feel that my voice has autonomy. That my opinion matters. Which is an absolute requirement to plan a future, or even just to validate own feelings.
这对我来说是一个全新的领域，让我感到既害怕又兴奋。 30 多年来，我第一次感到自己的声音具有自主性。我的意见很重要。这是计划未来的绝对要求，甚至只是为了验证自己的感受。

It might still take a while for me to naturally assert my needs and desires in conversation. But now that I'm aware of the habit and recognize the trigger and routine, and most importantly know a "healthy" routine to replace the fawn response with, I expect rapid improvement in this area.
我可能还需要一段时间才能在谈话中自然地表达自己的需求和愿望。但现在我意识到了这个习惯并认识到了触发因素和常规，最重要的是知道了一个“健康”的常规来代替小鹿的反应，我希望在这方面得到快速改善。

Just thinking this way is a huge shift in perspective. Getting my needs met is no longer about "how do I convince this person to do what I want" (which is a side effect of fawning, still only considering the other persons feelings), but becomes "how can I express myself in a way that reflects what I want and need".
仅仅以这种方式思考就是一个巨大的视角转变。满足我的需求不再是“我如何说服这个人做我想做的事”（这是阿谀奉承的副作用，仍然只考虑对方的感受），而是“我如何以某种方式表达自己”这反映了我想要和需要的东西”。

If you have read this far, I'm sorry. It's all pretty mundane. There is a paradigm shift in my thinking process and journaling helps me validate it.
如果你已经读到这里，我很抱歉。这一切都很平凡。我的思维过程发生了范式转变，日记帮助我验证了这一点。

希望能尽快提供更多有趣好玩的内容！




Train Girl
火车女孩

by Marius Bakke — Wed 16 November 2022
作者：Marius Bakke — 2022 年 11 月 16 日，星期三

Growing up in trauma can make it difficult to appreciate the sheer beauty of nature. Survivors like myself tend to hyper-fixate on any potential sign of trouble and completely miss the nice things in life.
在创伤中长大可能会让人难以欣赏大自然的纯粹之美。像我这样的幸存者往往会过分关注任何潜在的麻烦迹象，而完全错过生活中的美好事物。

We can also be afraid of making mistakes, to the point where we'll rather sacrifice our own well-being in the name of avoiding blame and shame.
我们也可能害怕犯错，以至于我们宁愿以避免责备和羞耻的名义牺牲自己的幸福。

Once I was on a nearly full train from Cardiff to Birmingham. I had a two-seater for myself, and across the aisle was a nice looking lady occupying another two-seater. We had some eye contact and there was clearly a mutual interest.
有一次我坐上了从加的夫到伯明翰的几乎满载的火车。我自己有一个双人座，过道对面是一位漂亮的女士，她坐在另一个双人座上。我们有一些眼神交流，显然有共同的兴趣。

When a couple entered at an intermediate station, I scooped over to the pretty lady so they could sit together, and I could heed my natural instinct. She clearly did not mind as we greet and smile. I contemplated for a while how to start a conversation and then...promptly fell asleep.
当一对夫妇从中间站进来时，我就凑到那位漂亮的女士身边，让他们坐在一起，我可以听从我的本能。当我们问候和微笑时，她显然不介意。我考虑了一会儿如何开始对话，然后……很快就睡着了。

I somehow woke up as the train pulled in to Birmingham station. She was clearly puzzled and probably disappointed by my sudden disinterest; and I was too embarassed to apologize. We both disembarked at Birmingham without exchanging a single word.
当火车驶入伯明翰车站时，我不知何故醒来。她显然对我突然不感兴趣感到困惑，可能还很失望。我太尴尬了，无法道歉。我们都在伯明翰下船，一句话也没说。

At a distance, I was keeping myself awake by romanticizing the idea of getting to know this woman, playing scenarios in my mind about how we could be soul mates and live happily ever after.
在远处，我通过浪漫化了解这个女人的想法来保持清醒，在我的脑海中播放关于我们如何成为灵魂伴侣并从此过上幸福生活的场景。

When I finally got the opportunity to talk, I got so obsessed with finding the "perfect words" that my body just shut down.
当我终于有机会说话时，我太着迷于寻找“完美的词”以至于我的身体刚刚停止。

I now know that words don't matter. Intent and actions do. After all, the "train girl" memory is one of love, hope, joy and ultimately disappointment and embarassment, all without ever saying a single word.
我现在知道，言语并不重要。意图和行动确实如此。毕竟，“火车女孩”的记忆是爱、希望、喜悦和最终的失望和尴尬，所有这些都没有说一个字。

Perhaps my biggest challenge now, as a recovering narcissist [abuse victim], is to make myself open and vulnerable. To put myself out there. Release decades of shame and fear and show up for myself every single day.
也许我现在最大的挑战，作为一个正在康复的自恋者 [虐待受害者]，是让自己开放和脆弱。把自己放在那里。释放数十年的羞耻和恐惧，每天都为自己现身。

It takes time. But I will get there. This blog is a good place for me to practice exposing myself without fear of embarassment, which often cripples me in person.
这需要时间。但我会到达那里。这个博客是我练习暴露自己而不用担心尴尬的好地方，这常常让我本人瘫痪。
