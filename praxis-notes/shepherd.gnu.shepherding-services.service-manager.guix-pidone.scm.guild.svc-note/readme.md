[site/services]: https://shepherding.services/ "The Shepherd herds system and user services, keeping you in the loop: you configure it and extend it in Scheme. // Shepherd 管理系统和用户服务，让你保持信息同步：你可以通过 Scheme 进行配置和扩展。 /// The Shepherd is a service manager written in Guile Scheme that looks after the herd of daemons running on the system. It can be used as an “init” system (PID 1) and also by unprivileged users to manage per-user daemons. It supports several daemon startup mechanisms—including inetd, systemd-style socket activation, and timers—and comes with a handy collection of services: syslog, log rotation, and more. The Shepherd is configured in Guile Scheme and can be extended in the same language. It builds on a simple memory-safe and callback-free programming model. // 牧羊人是一个用 Guile Scheme 编写的服务管理器，负责管理系统中运行的守护进程群。它可以用作“init”系统（PID 1），也可以由无特权的用户来管理每个用户的守护进程。它支持多种守护进程启动机制，包括 inetd、systemd 风格的套接字激活和计时器，并提供了一系列方便的服务：syslog、日志轮转等。Shepherd 使用 Guile Scheme 进行配置，并且可以在同一种语言中扩展。它基于一个简单的内存安全且无回调的编程模型。 /// The Shepherd is developed jointly with Guix. It runs as PID 1 on Guix System and manages user services for Guix Home. // 《GNU Shepherd》是与 Guix 共同开发的。它在 Guix System 上以 PID 1 运行，并管理 Guix Home 的用户服务。 /// The herd command interacts with a running shepherd process; as a user or system administrator, it is your entry point to service management // herd 命令与正在运行的 shepherd 进程交互；作为用户或系统管理员，它是服务管理的入口点 /// From writing your configuration to extending the Shepherd, and from extending it to contributing to it, there’s only a small step. An entire programming interface is at your disposal; if you’re adventurous, you can even fiddle with a running shepherd! Concurrency in shepherd is made simple by following the concurrent sequential processes (CSP) model using Fibers. Surely, that makes it the coolest init system to hack on. // 从编写你的配置到扩展 Shepherd，再到扩展它以贡献给它，只有一小步之遥。整个编程接口都为你所用；如果你喜欢冒险，甚至可以随意摆弄正在运行的 shepherd ！在 shepherd 中，通过使用 Fibers 采用并发顺序进程（CSP）模型，并发变得简单。这当然让它成为最适合进行黑客操作的 init 系统。 /// Licensing // 许可 /// The Shepherd is free software; you can redistribute it and/or modify it under the terms of the GNU General Public License, version 3 or any later version. // 《The GNU Shepherd》是自由软件；您可以在 GNU 通用公共许可证第 3 版或任何后续版本条款下重新分发和/或修改它。 /// History // 历史 /// The Shepherd was formerly known as GNU dmd. dmd was initially developed by Wolfgang Jährling, with input from other GNU hackers. It was first released in 2003, making it one of the first init systems departing from the venerable System V approach. Its development was then closely related to the GNU Hurd, although it has always supported GNU/Linux as well. // 牧羊人（Shepherd）以前被称为 GNU dmd。dmd 最初由 Wolfgang Jährling 开发，并得到了其他 GNU 黑客的输入。它于 2003 年首次发布，是首批脱离古老 System V 方法的开源初始化系统之一。其开发与 GNU Hurd 密切相关，尽管它一直也支持 GNU/Linux。 /// After a looong period of freeze, it was revived in 2013. It saw its first positively-numbered release in December 2013. It has since been used as the init system (PID 1) of Guix System. // 在漫长的冻结期后，它在 2013 年被重新激活。2013 年 12 月，它发布了第一个正数版本。从那时起，它被用作 Guix System 的初始化系统（PID 1）。 /// In January 2016, GNU dmd was renamed to GNU Shepherd to avoid confusion with a same-named project. // 2016 年 1 月，GNU dmd 被更名为 GNU Shepherd，以避免与一个同名的项目混淆。 /// Credits // 鸣谢 /// The Shepherd’s logo was designed by Luis Felipe López Acevedo.
