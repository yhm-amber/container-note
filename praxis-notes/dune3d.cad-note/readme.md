[src/gh]: https://github.com/dune3d/dune3d.git "(GPL-3.0) (Languages: C 85.3%, C++ 14.2%, GLSL 0.2%, Python 0.1%, Meson 0.1%, Shell 0.1%) 3D CAD application // 3D CAD 应用程序 /// Motivation // 动机 /// So why another open-source 3D CAD application when FreeCAD and Solvespace exist? My primary use case for 3D CAD is designing 3D-printed enclosures for my electronics projects. I often found myself procrastinating designing the enclosure and attributed that to my dissatisfaction with the available open source 3D CAD applications. // 那么，当 FreeCAD 和 Solvespace 已经存在时，为什么还要另一个开源 3D CAD 应用？我使用 3D CAD 的主要用例是为我的电子项目设计 3D 打印的外壳。我经常发现自己拖延设计外壳，并将这归因于我对现有的开源 3D CAD 应用的不满。 /// While FreeCAD technically does everything I need, the way it's implemented isn't quite to my liking. My biggest pain points with it are the modal sketcher that only works in 2D, no constraints in 3D for extrusions and the perils of referencing things in the design. // 虽然 FreeCAD 在技术上能满足我的所有需求，但其实现方式并不完全符合我的期望。我对 FreeCAD 最大的痛点在于：仅支持 2D 的模态草图器、3D 拉伸没有约束功能，以及设计中对引用元素的风险。 /// Solvespace on the other hand gets the workflow part right, but falls short by not importing STEP and the geometry kernel not supporting chamfers and fillets. // 另一方面，Solvespace 在流程方面做得很好，但缺点是无法导入 STEP 文件，且几何核心不支持倒角和圆角。 /// Having solved the similar problem for PCB CAD by developing Horizon EDA, I began pondering whether I could pull off the same thing for 3D CAD. After all, what does it take to make a 3D CAD? // 在通过开发 Horizon EDA 解决了 PCB CAD 的类似问题后，我开始思考是否可以将同样的方法应用于 3D CAD。毕竟，开发一款 3D CAD 需要什么？ /// - Geometry kernel to do extrusions, intersections, chamfers, etc.: While it's not a nice library to work with, Open CASCADE is the only viable choice if we want to have STEP import/export and fillets/chamfers. Fortunately, I have some experience with it from dealing with STEP files in Horizon EDA. // 几何核心用于执行拉伸、交集、倒角等操作：虽然 Open CASCADE 不是一个易于使用的库，但如果我们需要 STEP 导入/导出和圆角/倒角功能，它是唯一可行的选择。幸运的是，我在处理 Horizon EDA 中的 STEP 文件时积累了一些经验。 /// - 3D viewport: Obviously, we need a way to put 3D geometry on screen, zoom/pan and select things. The 3D preview in Horizon EDA already does all of this, so I have a well-understood codebase I can reuse. // 3D 视图：显然，我们需要一种方法将 3D 几何图形显示在屏幕上，进行缩放/平移和选择对象。Horizon EDA 中的 3D 预览已经实现了所有这些功能，因此我有一个可以重用的成熟代码库。 /// - Constraint solver: Unlike with Horizon EDA where things just stay where you last moved them, in 3D CAD, it's commonplace to specify where things go by means of constraints that need to be solved. Turns out that Solvespace's solver is available as a library1, so that part's also covered. // 约束求解器：与 Horizon EDA 中物体只是停留在最后一次移动的位置不同，在 3D CAD 中，通常通过需要求解的约束来指定物体的位置。结果是 Solvespace 的求解器可以作为库 1 使用，因此这部分也已涵盖。 /// - Editor infrastructure: Last but not least, we need code that takes care of the tools, undo/redo and all of the other bits and pieces that make up an interactive editor. While there'll be some differences, I felt confident that I could reuse and adapt the interactive manipulator from Horizon EDA. // 编辑器基础设施：最后但同样重要的是，我们需要处理工具、撤销/重做以及其他构成交互式编辑器的各种细节的代码。虽然会有一些差异，但我确信可以重用并调整 Horizon EDA 中的交互式操作器。 /// - With all of the building blocks available, I set out to glue them together to form a 3D CAD application. About three months later, it's somewhat presentable. // 在所有可用模块都准备就绪后，我开始将它们整合起来，构建一个 3D CAD 应用程序。大约三个月后，它已经可以勉强使用了。"
