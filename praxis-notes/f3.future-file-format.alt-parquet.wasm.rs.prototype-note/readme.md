[paper.doi/dl.acm.org]: https://dl.acm.org/doi/10.1145/3749163 "Columnar storage formats are the foundation for modern data analytics systems. The proliferation of open-source file formats (i.e., Parquet, ORC) allows seamless data sharing across disparate platforms. However, these formats were created over a decade ago for hardware and workload environments that are much different from today. Although these formats have incorporated some updates to their specification to adapt to these changes, not all deployments support those modifications, and too often systems cannot overcome the formats' deficiencies and limitations without a rewrite. // 列式存储格式是现代数据分析系统的基石。开源文件格式的普及（例如 Parquet、ORC）使得跨不同平台的数据共享变得无缝。然而，这些格式是在十多年前为与当今截然不同的硬件和工作负载环境创建的。尽管这些格式在其规范中已经包含了一些更新以适应这些变化，但并非所有部署都支持这些修改，而且系统往往无法在不重写的情况下克服这些格式的缺陷和局限性。 /// In this paper, we present the Future-proof File Format (F3) project. It is a next-generation open-source file format with interoperability, extensibility, and efficiency as its core design principles. F3 obviates the need to create a new format every time a shift occurs in data processing and computing by providing a data organization structure and a general-purpose API to allow developers to add new encoding schemes easily. Each self-describing F3 file includes both the data and meta-data, as well as WebAssembly (Wasm) binaries to decode the data. Embedding the decoders in each file requires minimal storage (kilobytes) and ensures compatibility on any platform in case native decoders are unavailable. To evaluate F3, we compared it against legacy and state-of-the-art open-source file formats. Our evaluations demonstrate the efficacy of F3's storage layout and the benefits of Wasm-driven decoding. // 在这篇论文中，我们介绍了 Future-proof File Format (F3) 项目。它是一种下一代开源文件格式，其核心设计原则是互操作性、可扩展性和效率。F3 通过提供一种数据组织结构和通用 API，使开发者能够轻松添加新的编码方案，从而避免了每次数据处理和计算发生转变时都需要创建新格式的需求。每个自我描述的 F3 文件都包含数据、元数据以及用于解码数据的 WebAssembly (Wasm) 二进制文件。将解码器嵌入每个文件需要极小的存储空间（千字节），并确保在原生解码器不可用时仍能在任何平台上兼容。为了评估 F3，我们将其与传统的和最先进的开源文件格式进行了比较。我们的评估展示了 F3 存储布局的有效性以及 Wasm 驱动解码的优势。"
