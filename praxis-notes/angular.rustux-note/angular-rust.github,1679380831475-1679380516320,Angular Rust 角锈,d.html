Overview概述

Angular Rust角锈

Angular Rust is a high productivity, platform-agnostic frontend framework for the Rust language. It now supports desktop and web development. Angular Rust currently uses Clutter for desktop development and WebAssembly for web development. We are planning to add support for mobile development.Angular Rust 是 Rust 语言的高生产力、平台无关的前端框架。它现在支持桌面和网络开发。 Angular Rust 目前使用 Clutter 进行桌面开发，使用 WebAssembly 进行 Web 开发。我们计划增加对移动开发的支持。

  • Simple starting point to development.简单的开发起点。
  • Utilize OpenGL >=1.4, OpenGL ES 1.1 and OpenGL ES 2.0.使用 OpenGL >=1.4、OpenGL ES 1.1 和 OpenGL ES 2.0。
  • Transparently utilize Cairo and HTML5 Canvas.透明地使用 Cairo 和 HTML5 Canvas。
  • Have an advanced animation features.拥有先进的动画功能。
  • Responsive and mobile device compatible.响应和移动设备兼容。
  • Customize and extend friendly.自定义和扩展友好。

Get Started开始

Lets write simple program, old school call it Hello Angular Rust.让我们编写简单的程序，老派称之为 Hello Angular Rust。

I am using Ubuntu for this tutorial. For Windows and MacOSX, the steps are not very different, but I will write the instructions a little later.我在本教程中使用 Ubuntu。对于 Windows 和 MacOSX，步骤差别不大，但稍后我会写说明。

There are some requirements for installed packages.对已安装的软件包有一些要求。

$ sudo apt-get install libcogl-dev libclutter-1.0-dev libpango1.0-dev

you may also need to run sudo apt-get update before installing packages. 您可能还需要在安装软件包之前运行 sudo apt-get update。

Just create new project. I prefer to create folder for my new project and then initialize project. So:只需创建新项目。我更喜欢为我的新项目创建文件夹，然后初始化项目。所以：

$ mkdir justapp
$ cd justapp
$ cargo init --bin

Cargo has created a folder structure as shown below:Cargo 创建了如下所示的文件夹结构：

Then add Angular Rust dependencies to your project. The UX Framework is the main starting point for getting started with developing Angular Rust applications. It allows you to shorten the path for accessing Angular Rust components and allows you to maintain consistent internal Angular Rust dependencies. Let's do it from the terminal using cargo-edit. 然后将 Angular Rust 依赖项添加到您的项目中。 UX 框架是开始开发 Angular Rust 应用程序的主要起点。它允许您缩短访问 Angular Rust 组件的路径，并允许您保持一致的内部 Angular Rust 依赖项。让我们使用 cargo-edit 从终端来完成它。

If you haven't installed cargo-edit, just cargo install cargo-edit如果您还没有安装 cargo-edit，只需 cargo install cargo-edit

$ cargo add ux-framework

Replace the code in the main.rs file替换main.rs文件中的代码

use ux::prelude::*;

#[derive(Default, Application)]
struct Application;

impl Application {
    fn new() -> Self {
        Default::default()
    }
}

fn main() {
    Application::run();
}

Let's run it from terminal:让我们从终端运行它：

$ cargo r

Oooh ... What's going on?喔喔……怎么回事？

You just launched the application and it works, but nothing happens. Let's make it more visible to see our application. 您刚刚启动了该应用程序并且它可以运行，但没有任何反应。让我们更清楚地看到我们的应用程序。

press Ctrl+C in terminal to exit our application 在终端中按 Ctrl+C 退出我们的应用程序

Add this line closer to the beginning of our source. This will allow us to use the window in our application. 将这一行添加到我们源代码的开头。这将允许我们在我们的应用程序中使用该窗口。

use ux::Window;

I am assuming that the window must belong to our application. So let's change the Application struct as shown below.我假设该窗口必须属于我们的应用程序。因此，让我们更改 Application 结构，如下所示。

#[derive(Default, Application)]
struct Application {
    window: Window,
}

If we run it again, we will see the same picture as before - nothing is happening. Let's configure our application window and show it. Change the new method of our application as shown bellows.如果我们再次运行它，我们将看到与之前相同的画面——什么也没有发生。让我们配置我们的应用程序窗口并显示它。如下所示更改我们应用程序的新方法。

fn new() -> Self {
    let app: Self = Default::default();
    app.window
        .show()
        .set_window_size(512, 512)
        .set_resizable(true)
        .set_title("Sample window")
        .connect_destroy(move |_win| {
            Application::quit()
        });
    
    app.window.set_background_color(Some(color::TEAL_9));
    
    app
}

It's time to see our app.是时候看看我们的应用程序了。

$ cargo r

Let me explain what we have written in the new method of our application.让我解释一下我们在应用程序的新方法中编写的内容。

  1. We created the structure of our application as before, but saved it in a variable to access the application window.我们像以前一样创建了应用程序的结构，但将其保存在一个变量中以访问应用程序窗口。
let app: Self = Default::default();
  1. Below we have customized our window and shown it. We have set the initial window size and allowed the application window to be resized. Then we changed the title of our window.下面我们定制了我们的窗口并显示了它。我们已经设置了初始窗口大小并允许调整应用程序窗口的大小。然后我们更改了窗口的标题。
app.window
    .show()
    .set_window_size(512, 512)
    .set_resizable(true)
    .set_title("Sample window")
  1. I would like the application to exit correctly when the window is closed. For this, we added a handler for the "window-destroy" event to properly exit our application.我希望应用程序在窗口关闭时正确退出。为此，我们为“window-destroy”事件添加了一个处理程序，以正确退出我们的应用程序。
...
    .connect_destroy(move |_win| {
        Application::quit()
    });
  1. We painted our window in TEAL color because - Rake Likes This. 我们将窗户涂成蓝绿色，因为 - Rake 喜欢这个。
app.window.set_background_color(Some(color::TEAL_9));
  1. Finally, we return an instance of our application, by last line app.最后，我们通过最后一行 app 返回我们应用程序的一个实例。
