[goal/site]: https://rhombus-lang.org/goal.html "Modern programming languages reflect a consensus on basic constructs, including functional abstraction, lexically scoped variables, closures, objects, and pattern matching. Why, then, are we creating yet another general-purpose programming language? // 现代编程语言在基本结构上达成共识，包括函数抽象、词法作用域变量、闭包、对象和模式匹配。那么，我们为什么要创建另一种通用编程语言呢？ /// Beyond the basics, there are still more good ideas for programming constructs than can fit in any one language specification. Furthermore, specific domains benefit from language support that is tailored to the domain. Language extensibility helps to balance the competing goals of a manageable language size versus fit-to-purpose for a wide range of tasks. // 除了基础知识之外，仍然有更多优秀的编程结构想法无法包含在任何一种语言规范中。此外，特定领域受益于针对该领域量身定制的语言支持。语言可扩展性有助于平衡可管理语言大小与广泛任务适用性之间的竞争目标。 /// Many newer languages include a macro system to enable extensibility, but few would argue that the new batch of macro systems have achieved the expressiveness and fluidity of macros as they exist within the Lisp tradition, which includes Racket. At the same time, that expressiveness has been difficult to detangle from Lisp’s minimalistic, parenthesis-oriented notation. // 许多较新的语言包含宏系统以实现可扩展性，但很少有人会认为新一代宏系统达到了 Lisp 传统（包括 Racket）中宏的表达性和流畅性。同时，这种表达性很难与 Lisp 的极简主义、括号导向的记法分离开来。 /// Rhombus is designed to be // 菱形设计旨在 /// - approachable and easy to use for everyday purposes (that do not need macros), which in part means a conventional syntax; and // 易于日常使用（不需要宏），这意味着它具有传统的语法；并且 /// - as extensible as Racket, while making Racket's state-of-the-art facilities more consistent and accessible to a wide audience. // 与 Racket 一样可扩展，同时使 Racket 的最新功能更加一致和易于广大用户使用。"
