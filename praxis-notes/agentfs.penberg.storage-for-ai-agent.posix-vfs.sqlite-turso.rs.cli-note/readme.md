[src/gh]: https://github.com/penberg/agentfs.git "(Languages: Rust 73.3%, TypeScript 23.2%, Shell 3.0%, C 0.5%) The filesystem for agents // 给智能特工的文件系统 /// AgentFS is a filesystem designed specifically for AI agents. Just as traditional filesystems provide file and directory abstractions for applications, AgentFS provides the storage abstractions that AI agents need. // AgentFS 是专为 AI 代理设计的文件系统。正如传统文件系统为应用程序提供文件和目录抽象一样，AgentFS 提供了 AI 代理所需的存储抽象。 /// At the heart of AgentFS is the [agent filesystem](https://github.com/penberg/agentfs/blob/main/SPEC.md 'Agent Filesystem Specification // 代理文件系统规范') - a complete SQLite-based storage system for agents implemented using Turso (. gh:tursodatabase/turso.git). It combines three essential components: a POSIX-like virtual filesystem for files and directories, a key-value store for agent state and context, and an audit trail tool for debugging and analysis. Everything an agent does - every file it creates, every piece of state it stores, every tool it invokes - lives in a single SQLite database file. // AgentFS 的核心是代理文件系统——一个基于 SQLite 的完整存储系统，使用 Turso 实现。它结合了三个关键组件：一个类似 POSIX 的虚拟文件系统用于文件和目录，一个键值存储用于代理状态和上下文，以及一个用于调试和分析的审计工具。代理所做的每一件事——创建的每个文件、存储的每段状态、调用的每个工具——都存在于一个单一的 SQLite 数据库文件中。 /// The Agent Filesystem Specification defines a SQLite schema for representing agent filesystem state. The specification consists of three main components: // Agent Filesystem 规范定义了一个 SQLite 模式，用于表示 agent 文件系统状态。该规范由三个主要组件组成： /// 1. **Tool Call Audit Trail**: Captures tool invocations, parameters, and results for debugging, auditing, and performance analysis // 工具调用审计追踪：捕获工具调用、参数和结果，用于调试、审计和性能分析 /// 2. **Virtual Filesystem**: Stores agent artifacts (files, documents, outputs) using a Unix-like inode design with support for hard links, proper metadata, and efficient file operations // 虚拟文件系统：使用类似 Unix 的 inode 设计存储代理工件（文件、文档、输出），支持硬链接、正确的元数据和高效的文件操作 /// 3. **Key-Value Store**: Provides simple get/set operations for agent context, preferences, and structured state that doesn't fit into the filesystem model // 键值存储：为代理上下文、偏好设置和不符合文件系统模型的有序状态提供简单的 get/set 操作 /// All timestamps in this specification use Unix epoch format (seconds since 1970-01-01 00:00:00 UTC). // 本规范中的所有时间戳均使用 Unix 纪元格式（自 1970-01-01 00:00:00 UTC 以来的秒数）。 /// AgentFS provides four main components: // AgentFS 提供了四个主要组件： /// - **CLI** (. gh: penberg/agentfs.git @main/MANUAL.md) - Command-line interface for managing agents // CLI - 用于管理代理的命令行界面 /// - **SDK** - TypeScript and Rust libraries for programmatic access // SDK - TypeScript 和 Rust 库，用于程序化访问 /// - **Sandbox** - Linux-compatible execution environment // Sandbox - 兼容 Linux 的执行环境 /// - **Specification** (. gh: penberg/agentfs.git @main/SPEC.md) - SQLite-based filesystem architecture // Specification - 基于 SQLite 的文件系统架构"
