[site/ghio]: https://rticulate.github.io/import/ "The import package is intended to simplify the way in which functions from external packages or modules are made available for use in R scripts. Learn more on the package website, by reading `vignette('import')`, or using the help (`?import::from`). // import 包旨在简化外部包或模块中的函数在 R 脚本中的使用方式。在包网站、阅读 `vignette('import')` 或使用 help (`?import::from`) 中了解更多信息。 /// The typical way of using functionality exposed by a package in R scripts is to load (and attach) the entire package with library() (or require()). This can have the undesirable effect of masking objects in the user’s search path and can also make it difficult and confusing to identify what functionality comes from which package when using several library statements. // 在 R 脚本中使用包提供的功能通常是通过 library() （或 require() ）加载（并附加）整个包。这可能会掩盖用户搜索路径中的对象，并且在使用多个 library 语句时，也难以和混淆地识别哪些功能来自哪个包。 /// The import package provides a simple alternative, allowing the user specify in a concise way exactly which objects. For example, the Hmisc package exposes over four hundred functions. Instead of exposing all of those functions, someone who only needs access to, say the impute() and the nomiss() functions, can import those functions only. // import 包提供了一个简单的替代方案，允许用户以简洁的方式指定具体哪些对象。例如， Hmisc 包公开了四百多个函数。与其公开所有这些函数，只需要访问，比如 impute() 和 nomiss() 函数的人，可以只导入这些函数。"

Furthermore the imported objects are not placed in the current environment. // 一种从包和 R 模块中导入对象的替代机制。该语法允许以表达的方式用一条命令导入多个对象。import 包在使用 library（或 require）和直接（单个对象）导入之间架起了一部分桥梁。此外，导入的对象不会放置在当前环境中。 (src: gh:rticulate/import.git)"
