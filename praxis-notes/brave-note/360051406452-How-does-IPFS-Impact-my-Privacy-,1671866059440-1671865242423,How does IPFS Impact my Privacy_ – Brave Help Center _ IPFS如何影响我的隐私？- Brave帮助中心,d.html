How does IPFS Impact my Privacy? FollowIPFS如何影响我的隐私？

The InterPlanetary File System (IPFS) is a protocol enabling a globally distributed, peer-to-peer network for storing and accessing files, websites, applications and data. IPFS uses content addressing, rather than traditional location-based addressing, to uniquely identify each file in a global namespace connecting every computing device that participates in the network.行星际文件系统（IPFS）是一个协议，它实现了一个全球分布的、点对点的网络，用于存储和访问文件、网站、应用程序和数据。IPFS使用内容寻址，而不是传统的基于位置的寻址，在一个连接每个参与网络的计算设备的全球命名空间中唯一地识别每个文件。

As a Brave user, you can become part of the IPFS network by installing and starting an IPFS node within the Brave browser itself. This means that when Brave detects IPFS-hosted resources, it will automatically use your browser’s IPFS node to retrieve them. Because IPFS is a public, peer-to-peer network, when you enable IPFS support in Brave, you should be aware of the following privacy considerations:作为一个Brave用户，你可以通过在Brave浏览器中安装和启动一个IPFS节点来成为IPFS网络的一部分。这意味着当Brave检测到IPFS托管的资源时，它将自动使用你的浏览器的IPFS节点来检索它们。由于IPFS是一个公共的，点对点的网络，当你在Brave中启用IPFS支持时，你应该注意以下隐私问题：

  1. When your device accesses content over IPFS, it also temporarily becomes a host for the content until your node’s local cache is cleared (this amount of time can vary, but the cache is checked hourly and will clear if it reaches 90% of its default 1GB limit). This means that if you visit a page over IPFS, you agree to temporarily be a host of any IPFS resources that it loads.当你的设备通过IPFS访问内容时，它也会暂时成为该内容的主机，直到你的节点的本地缓存被清除（这个时间可能会有所不同，但缓存是每小时检查一次，如果达到其默认的1GB限制的90%，将被清除）。这意味着，如果你通过IPFS访问一个页面，你同意暂时成为它所加载的任何IPFS资源的主机。
  2. Every IPFS node on the network has a unique public identifier, known as a PeerID. Since PeerIDs are long-lived identities, it’s possible for someone to look up your PeerID in the distributed hash table (DHT) that stores public IPFS metadata and track your IP address over time.网络上的每个IPFS节点都有一个独特的公共标识符，被称为PeerID。由于PeerID是长期存在的身份，有人有可能在存储公共IPFS元数据的分布式哈希表（DHT）中查找你的PeerID，并在一段时间内跟踪你的IP地址。
  3. Similarly, because that DHT also contains metadata about the content that is requested or provided by PeerIDs, it’s possible for someone observing the DHT to learn what content your node (and, potentially, your IP address) is requesting from or providing to the rest of the IPFS network.同样，由于DHT也包含了关于由PeerID请求或提供的内容的元数据，因此观察DHT的人有可能了解到你的节点（以及可能是你的IP地址）从IPFS网络的其他部分请求或提供什么内容。

  4. If you want to avoid running a local node, you can load IPFS through the public gateway. However, this allows the gateway to see your IPFS browsing activity. This public gateway can see and even log your IP address.如果你想避免运行一个本地节点，你可以通过公共网关加载IPFS。然而，这允许该网关看到你的IPFS浏览活动。这个公共网关可以看到，甚至可以记录你的IP地址。

  5. A passive adversary can also learn that you are on the Brave browser.一个被动的对手也可以了解到你在使用Brave浏览器。

  6. In general, network requests from your local IPFS node will NOT use your browser proxy settings, for instance if you are using a VPN browser extension. If you are using a system proxy, it's also possible the node will send some requests that do not go through the system proxy.一般来说，来自你本地IPFS节点的网络请求不会使用你的浏览器代理设置，例如，如果你使用VPN浏览器扩展。如果你使用的是系统代理，也有可能该节点会发送一些不通过系统代理的请求。

You can learn more about IPFS and privacy, including strategies for mitigating these privacy considerations, in the official IPFS docs.你可以在官方IPFS文档中了解更多关于IPFS和隐私的信息，包括减轻这些隐私考虑的策略。
