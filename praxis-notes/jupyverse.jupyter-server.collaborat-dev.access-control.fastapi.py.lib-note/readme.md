[site/ghio]: https://jupyter-server.github.io/jupyverse/ "Jupyverse is a FastAPI-based Jupyter server. It can be used to run JupyterLab and other web clients in different scenarios: // Jupyverse 是一个基于 FastAPI 的 Jupyter 服务器。它可用于在不同场景中运行 JupyterLab 和其他 Web 客户端： /// - On your personal machine: that's the most common use case, where you want to do interactive computing, most often running notebooks, in your local environment. Although JupyterLab runs in the browser, it really feels like a desktop application. // 在你的个人电脑上：这是最常见的使用场景，你希望在本地环境中进行交互式计算，通常运行笔记本。尽管 JupyterLab 在浏览器中运行，但它确实感觉像是一个桌面应用程序。 /// - Collaboratively: users work on the same document in real time, connected to the same server. That means they are on the same private network (LAN), or the server is hosted on the Internet. The user experience is similar to Google Docs. // 协同工作：用户可以实时共同编辑同一份文档，他们连接到同一台服务器。这意味着他们处于同一个私有网络（局域网）中，或者服务器托管在互联网上。用户体验与 Google Docs 类似。 /// - With access control: this becomes necessary when working collaboratively, as you don't want everybody to be able to see or change any thing. You want to know who users are and give them more or less permissions. // 访问控制：在协同工作时，这变得必要，因为你不想让每个人都能够查看或更改任何内容。你想知道用户是谁，并给予他们不同程度的权限。"
