[fmt.intro/openai]: https://cookbook.openai.com/articles/openai-harmony "OpenAI Harmony Response Format // OpenAI 和谐响应格式 /// Role & Purpose /// - system:	A system message is used to specify reasoning effort, meta information like knowledge cutoff and built-in tools // 系统消息用于指定推理工作量和元信息，如知识截止日期和内置工具 /// - developer:	The developer message is used to provide information about the instructions for the model (what is normally considered the “system prompt”) and available function tools // 开发者消息用于提供有关模型指令的信息（通常被认为是“系统提示”）以及可用的函数工具 /// - user:	Typically representing the input to the model // 通常代表输入到模型的内容 /// - assistant:	Output by the model which can either be a tool call or a message output. The output might also be associated with a particular “channel” identifying what the intent of the message is. // 模型输出的内容，可以是工具调用或消息输出。输出还可能与特定的“通道”相关联，以识别消息的意图。 /// - tool:	Messages representing the output of a tool call. The specific tool name will be used as the role inside a message. // 表示工具调用输出的消息。具体的工具名称将作为消息中的角色使用。 /// These roles also represent the information hierarchy that the model applies in case there are any instruction conflicts: system > developer > user > assistant > tool // 这些角色还表示模型在存在任何指令冲突时应用的信息层级： system > developer > user > assistant > tool /// Channel & Purpose /// - final:	Messages tagged in the final channel are messages intended to be shown to the end-user and represent the responses from the model. // 最终频道中标记的消息是打算展示给最终用户的消息，代表模型的响应。 /// - analysis:	These are messages that are being used by the model for its chain of thought (CoT). Important: Messages in the analysis channel do not adhere to the same safety standards as final messages do. Avoid showing these to end-users. // 这些是模型用于其思维链（CoT）的消息。重要提示：分析频道中的消息不遵循最终消息的相同安全标准。避免向最终用户展示这些消息。 /// - commentary:	Any function tool call will typically be triggered on the commentary channel while built-in tools will normally be triggered on the analysis channel. However, occasionally built-in tools will still be output to commentary. Occasionally this channel might also be used by the model to generate a preamble to calling multiple functions. // 任何函数工具调用通常会在 commentary 频道触发，而内置工具通常会在 analysis 频道触发。然而，偶尔内置工具仍会输出到 commentary 。偶尔这个频道也可能被模型用于生成调用多个函数的前置文本。"
