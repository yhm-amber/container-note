[guide]: https://asdf-vm.com/guide/ "asdf is a tool version manager. All tool version definitions are contained within one file (.tool-versions) which you can check in to your project's Git repository to share with your team, ensuring everyone is using the exact same versions of tools. // asdf 是一个工具版本管理器。所有工具版本定义都包含在一个文件（ .tool-versions ）中，你可以将其检入到项目的 Git 仓库中与团队共享，确保每个人都在使用完全相同的工具版本。 /// The old way of working required multiple CLI version managers, each with their distinct API, configurations files and implementation (e.g. $PATH manipulation, shims, environment variables, etc...). asdf provides a single interface and configuration file to simplify development workflows, and can be extended to all tools and runtimes via a simple plugin interface. // 旧的工作方式需要多个 CLI 版本管理器，每个都有其独特的 API、配置文件和实现（例如 $PATH 操作、shims、环境变量等）。 asdf 提供了一个单一的接口和配置文件来简化开发工作流程，并且可以通过简单的插件接口扩展到所有工具和运行时。 /// How It Works // 工作原理 /// Once asdf core is set up with your Shell configuration, plugins are installed to manage particular tools. When a tool is installed by a plugin, the executables that are installed have shims created for each of them. When you try and run one of these executables, the shim is run instead, allowing asdf to identify which version of the tool is set in .tool-versions and execute that version. // 一旦 asdf 核心与你的 Shell 配置设置完成，就会安装插件来管理特定的工具。当一个工具被插件安装时，安装的可执行文件会为每个创建 shims。当你尝试运行其中一个可执行文件时，运行的是 shims 而不是实际的执行文件，这允许 asdf 识别 .tool-versions 中设置的工具版本并执行该版本。"
