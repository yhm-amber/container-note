Lunatic is an Erlang-inspired runtime for WebAssemblyLunatic 是受 Erlang 启发的 WebAssembly 运行时

By combining the fault-tolerance and massive concurrency of Erlang with the capability-based security of WebAssembly, it creates a powerful programming model.通过将 Erlang 的容错和大规模并发与 WebAssembly 基于功能的安全性相结合，它创建了一个强大的编程模型。

Architecture建筑学

Lunatic is a single executable and runs on Windows, macOS and Linux. It spawns lightweight processes from WebAssembly modules that have their own heap/stack. These processes are preemptively scheduled on a multi-threaded executor.Lunatic 是一个单一的可执行文件，可在 Windows、macOS 和 Linux 上运行。它从具有自己的堆/堆栈的 WebAssembly 模块生成轻量级进程。这些进程被抢占式调度在多线程执行器上。

Processes进程

Lunatic's design is all about super lightweight processes. Processes are fast to create, have a small memory footprint and a low scheduling overhead. They are designed for massive concurrency.Lunatic 的设计完全是关于超轻量级流程。进程创建速度快，内存占用小，调度开销低。它们专为大规模并发而设计。

Isolation隔离

Lunatic sits between your operating system and the running processes. This allows it to decide on a per process basis what syscalls are going to be forwarded to the operating systems.Lunatic 位于您的操作系统和正在运行的进程之间。这允许它在每个进程的基础上决定将哪些系统调用转发给操作系统。

If you would like to run some untrusted code, you can. Just spawn it inside of a process without any privileges.如果你想运行一些不受信任的代码，你可以。只需在没有任何特权的情况下在进程内生成它。

Features特征

Open source开源

Lunatic is an open source project licensed under the MIT/Apache license. You can check out the code on Github.Lunatic 是一个在 MIT/Apache 许可证下获得许可的开源项目。你可以在 Github 上查看代码。

Secure安全的

Lunatic builds on WebAssembly's security. We all use unaudited third-party libs that get deployed with our code, Lunatic can use capability based security to limit them.Lunatic 建立在 WebAssembly 的安全性之上。我们都使用与我们的代码一起部署的未经审计的第三方库，Lunatic 可以使用基于功能的安全性来限制它们。

Fast快速地

Lunatic uses Wasmtime to run Wasm and just-in-time compile your Wasm to machine code (using Cranelift).Lunatic 使用 Wasmtime 运行 Wasm 并及时将 Wasm 编译为机器代码（使用 Cranelift）。

Multi-lingual多种语言

We want to support all languages that compile to Wasm. Some are better suited for Lunatic's use-cases. Currently, we are focused on Rust and AssemblyScript.我们希望支持编译为 Wasm 的所有语言。有些更适合 Lunatic 的用例。目前，我们专注于 Rust 和 AssemblyScript。

No lock-in无锁定

You have the freedom to architecture your app in any way you want. Ideally you should be able to compile your existing app to Wasm and enjoy some benefits of Lunatic.您可以自由地以任何您想要的方式构建您的应用程序。理想情况下，您应该能够将现有应用程序编译为 Wasm 并享受 Lunatic 的一些好处。

Robust强壮的

Lunatic processes are completely isolated, with a per-process stack and heap. If a process crashes, it won't affect others in the runtime.Lunatic 进程是完全隔离的，每个进程都有一个栈和堆。如果一个进程崩溃，它不会影响运行时中的其他进程。
