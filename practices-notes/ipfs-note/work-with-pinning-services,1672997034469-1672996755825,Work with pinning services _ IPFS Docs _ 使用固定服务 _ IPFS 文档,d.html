# Work with remote pinning services# 使用远程固定服务

Depending on how you use IPFS, you might find it helpful to use a remote pinning service instead of, or in addition to, pinning files on your local IPFS node. Whether it happens remotely or locally, pinning an item in IPFS identifies it as something you always wish to keep available, exempting it from the routine garbage collection that IPFS does on infrequently-used items in order to efficiently manage storage space. Learn more about local pinning →根据您使用 IPFS 的方式，您可能会发现使用远程固定服务代替或补充将文件固定在本地 IPFS 节点上会很有帮助。无论是远程发生还是本地发生，在 IPFS 中固定一个项目都会将其标识为您始终希望保持可用的东西，从而使其免于 IPFS 对不常用项目进行的例行垃圾收集，以便有效地管理存储空间。了解有关本地固定的更多信息 →

If you've got just one local IPFS node that's always running, local pinning may be all you need to ensure your important items are persisted and never garbage-collected. However, using a remote pinning service — or creating your own — might be useful to you if:如果您只有一个始终在运行的本地 IPFS 节点，则可能只需要本地固定就可以确保您的重要项目持久化并且永远不会被垃圾收集。但是，在以下情况下，使用远程固定服务（或创建您自己的固定服务）可能对您有用：

  • Your local node isn't always online, but you need items to be consistently available.您的本地节点并不总是在线，但您需要项目始终可用。
  • You'd like to keep a persistent backup of your local node's files somewhere else.您希望在其他地方保留本地节点文件的持久备份。
  • You don't have all the disk space you need on your local node.您的本地节点上没有所需的所有磁盘空间。
  • You run more than one IPFS node, and would like to use one of them as a "personal pinning service" as your preferred location for permanent storage.您运行多个 IPFS 节点，并希望将其中一个节点用作“个人固定服务”作为永久存储的首选位置。

There are a number of commercial pinning services that make it easy for you to purchase pinning capacity for your important files, some of which include Pinata, Temporal, Crust, Infura, and others. Each of these third-party services has its own unique interface for pinning files and managing those pins; this could include a GUI, an API, CLI commands, or other tooling.有许多商业固定服务可以让您轻松购买重要文件的固定容量，其中一些包括 Pinata、Temporal、Crust、Infura 等。这些第三方服务中的每一个都有自己独特的用于固定文件和管理这些固定的界面；这可能包括 GUI、API、CLI 命令或其他工具。

However, you don't need to learn new commands or tools if your pinning service of choice supports the vendor-agnostic IPFS Pinning Service API (opens new window) specification. Those services are supported within IPFS itself through the command line: ipfs pin remote --help.但是，如果您选择的固定服务支持与供应商无关的 IPFS 固定服务 API，则无需学习新命令或工具 （打开新窗口） 规格。 IPFS 本身通过命令行支持这些服务：ipfs pin remote --help。

As of January 2021, Pinata (opens new window) supports the IPFS Pinning Service API endpoint (opens new window), with more pinning services on the way! Learn how to create your own →截至 2021 年 1 月，Pinata （打开新窗口） 支持 IPFS Pinning Service API 端点 （打开新窗口） ，更多固定服务即将推出！了解如何创建自己的 →

# Use an existing pinning service# 使用现有的固定服务

There are two methods to add an existing pinning service to your IPFS installation:有两种方法可以将现有的固定服务添加到您的 IPFS 安装中：

  • IPFS Desktop or IPFS Web UIIPFS 桌面或 IPFS Web UI
  • IPFS CLIIPFS命令行

To add and use a remote pinning service directly in IPFS, you'll first need to have an account with that service.要直接在 IPFS 中添加和使用远程固定服务，您首先需要拥有该服务的帐户。

# IPFS Desktop or IPFS Web UI# IPFS 桌面或 IPFS Web UI

You can add your favorite pinning service(s) to IPFS Desktop/Web UI directly, enabling you to pin and unpin items from the Files screen in the same way as as you do local pins.您可以将您最喜欢的固定服务直接添加到 IPFS 桌面/Web UI，使您能够以与本地固定相同的方式从文件屏幕固定和取消固定项目。

# Adding a new pinning service# 添加新的固定服务

To add a new pinning service, open up IPFS Desktop or Web UI, navigate to the Pinning Services section of the Settings screen, and click the Add Service button:要添加新的固定服务，请打开 IPFS 桌面或 Web UI，导航到“设置”屏幕的“固定服务”部分，然后单击“添加服务”按钮：

Then, select your chosen pinning service from the modal that pops up. If the pinning service you'd like to add isn't listed in that modal, don't worry — you can add any remote pinning service that supports the IPFS Pinning Service API by clicking the Add a custom one link.然后，从弹出的模式中选择您选择的固定服务。如果您要添加的固定服务未在该模式中列出，请不要担心 — 您可以通过单击添加自定义链接来添加任何支持 IPFS 固定服务 API 的远程固定服务。

In the next screen, you’ll be asked for a few other details:在下一个屏幕中，系统会要求您提供一些其他详细信息：

Preconfigured service (eg. Pinata)预配置服务（例如 Pinata） Custom service客户服务
  • A nickname for this service. This can be helpful if, for example, you want to add two accounts from the same service.此服务的昵称。例如，如果您想从同一服务添加两个帐户，这会很有帮助。
  • The URL for your service's API endpoint.您服务的 API 端点的 URL。
    Note: This field only appears if you've selected a custom pinning service!注意：此字段仅在您选择了自定义固定服务时才会出现！
  • Your secret access token. This is the unique token provided to you by the pinning service — check its documentation for more info.您的秘密访问令牌。这是固定服务提供给您的唯一令牌——查看其文档以获取更多信息。
    To illustrate, example below shows which value should be copied from pinata.cloud/keys (opens new window)为了说明，下面的示例显示了应该从 pinata.cloud/keys 复制哪个值 （打开新窗口）

After you hit Save, you’ll see your new pinning service added to the Pinning Services section of your Settings screen.点击“保存”后，您会看到新的固定服务已添加到“设置”屏幕的“固定服务”部分。

# Using a pinning service# 使用固定服务

Now that you’re set up, you can pin or unpin files to your new pinning service directly from the Files screen. Just right-click any file or click the three dots action icon in the files list, and select Set pinning:设置完毕后，您可以直接从“文件”屏幕将文件固定或取消固定到新的固定服务。只需右键单击任何文件或单击文件列表中的三点操作图标，然后选择设置固定：

# IPFS CLI# IPFS命令行

Command-line users benefit from ipfs pin remote commands, which simplify remote pinning operations. The built-in pinning service API client executes all the necessary remote calls under the hood.命令行用户受益于 ipfs pin 远程命令，它简化了远程固定操作。内置固定服务 API 客户端在后台执行所有必要的远程调用。

# Adding a new pinning service# 添加新的固定服务

To add a new pinning service, use the following command:要添加新的固定服务，请使用以下命令：

$ ipfs pin remote service add nickname https://my-pin-service.example.com/api-endpoint myAccessToken
  • nickname is a unique name for this particular instantiation of a pinning service. This can be helpful if, for example, you want to add two accounts from the same service.nickname 是固定服务的这个特定实例的唯一名称。例如，如果您想从同一服务添加两个帐户，这会很有帮助。
  • https://my-pin-service.example.com/api-endpoint is the endpoint supplied to you by the pinning service. Check the service's documentation for more info.https://my-pin-service.example.com/api-endpoint 是固定服务提供给您的端点。查看服务的文档以获取更多信息。
  • myAccessToken is the unique secret token provided to you by the pinning service. Check the service's documentation for more info.myAccessToken 是固定服务提供给您的唯一秘密令牌。查看服务的文档以获取更多信息。

# Using a pinning service# 使用固定服务

Here are a few CLI commands to get you started. In all examples, replace nickname with the unique name you gave the pinning service when you added it.以下是一些可帮助您入门的 CLI 命令。在所有示例中，将昵称替换为您在添加固定服务时为其提供的唯一名称。

To pin a CID under under a human-readable name:将 CID 固定在人类可读的名称下：

$ ipfs pin remote add --service=nickname --name=war-and-peace.txt bafybeib32tuqzs2wrc52rdt56cz73sqe3qu2deqdudssspnu4gbezmhig4

To list successful pins:列出成功的引脚：

$ ipfs pin remote ls --service=nickname

To list all "pending" pins:列出所有“待定”引脚：

$ ipfs pin remote ls --service=nickname --status=queued,pinning,failed

For more commands and general help:有关更多命令和一般帮助：

$ ipfs pin remote --help

# Create your own pinning service# 创建自己的固定服务

Obviously you aren't limited to a static list of pre-approved services. Any remote pinning service compatible with the IPFS Pinning Service API (opens new window) can be added as a custom pinning service — which also means that you can create your own! This might be useful in circumstances like:显然，您不限于预先批准的服务的静态列表。任何与 IPFS Pinning Service API 兼容的远程 Pinning 服务 （打开新窗口） 可以添加为自定义固定服务——这也意味着您可以创建自己的固定服务！这可能在以下情况下有用：

  • Designating one of your own IPFS nodes to be a personal pinning service as a preferred location for permanent storage.将您自己的 IPFS 节点之一指定为个人固定服务，作为永久存储的首选位置。
  • Running a private pinning service for your friends or company.为您的朋友或公司运行私人固定服务。
  • Starting your own commercial pinning service.开始您自己的商业固定服务。

As noted above, your service must use the IPFS Pinning Service API (opens new window) in order to be interoperable with client behind ipfs pin remote commands.如上所述，您的服务必须使用 IPFS Pinning Service API （打开新窗口） 为了与 ipfs pin 远程命令后面的客户端互操作。

TIP提示

If you're interested in creating your own pinning service for your own personal or shared use, you can generate client and server from the OpenAPI spec (opens new window), reducing the development time, or reuse an existing solution (opens new window)如果您有兴趣为自己的个人或共享用途创建自己的固定服务，您可以从 OpenAPI 规范生成客户端和服务器 （打开新窗口） ，减少开发时间，或重用现有解决方案 （打开新窗口）

You may also wish to read continuing details on how the API is evolving in the Pinning Service API Spec GitHub repo (opens new window), and be part of the discussion on its further development!您可能还希望在固定服务 API 规范 GitHub 存储库中继续阅读有关 API 如何发展的详细信息 （打开新窗口） ，并参与其进一步发展的讨论！

If you'd like to make your custom pinning service available to every IPFS user, we welcome your submissions. Once you're ready to open the doors to the public, make a PR against the IPFS Web UI GitHub repo (opens new window) in order to add it to the default list of pinning services that are displayed in the Desktop/Web UI Settings screen, and one of the core maintainers will be in touch.如果你想让每个 IPFS 用户都可以使用你的自定义固定服务，我们欢迎你提交。一旦您准备好向公众敞开大门，就可以针对 IPFS Web UI GitHub 存储库进行 PR （打开新窗口） 为了将其添加到桌面/Web UI 设置屏幕中显示的默认固定服务列表中，核心维护者之一将与您联系。
